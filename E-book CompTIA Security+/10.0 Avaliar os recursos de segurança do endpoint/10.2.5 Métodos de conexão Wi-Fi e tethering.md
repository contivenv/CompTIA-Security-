# Security+ CertMaster Study (Versão em português) 7.0

Os dispositivos móveis geralmente são configurados para usar uma conexão Wi-Fi para dados, se estiver disponível. Se o usuário estabelecer uma conexão com uma rede corporativa que usa uma forte segurança WPA3, há um risco bastante baixo de espionagem ou de ataques de invasores intermediários. Os riscos do Wi-Fi vêm de usuários que se conectam a access points abertos ou possivelmente a um access point mal-intencionado que imita uma rede corporativa. Eles permitem que o proprietário do access point inicie diversos ataques, até mesmo comprometendo potencialmente sessões com servidores seguros (usando um ataque de falsificação de DNS, por exemplo).

## Redes de área pessoal (Personal Area Networks, PANs)

[[Redes de área pessoal (PANs)]] permitem a conectividade entre um dispositivo móvel e periféricos. Também é possível estabelecer redes ad hoc (ou ponto a ponto) entre dispositivos móveis ou entre dispositivos móveis e outros dispositivos de computação. Em termos de segurança corporativa, essas funções ponto a ponto geralmente devem ser desativadas. Um invasor pode explorar um dispositivo configurado incorretamente e realizar uma conexão em ponte com a rede corporativa.

## Wi-Fi Ad Hoc e Wi-Fi Direct

As estações wireless podem estabelecer conexões ponto a ponto entre si, em vez de usar um access point. Isso também pode ser chamado de [[rede ad hoc]], o que significa que a rede não é disponibilizada permanentemente. No entanto, não há suporte estabelecido baseado em padrões para redes ad hoc. A MITRE possui um projeto para permitir que smartphones Android se configurem em uma rede ad hoc ([mitre.org/research/technology-transfer/open-source-software/smartphone-ad-hoc-networking-span](https://www.mitre.org/research/technology-transfer/open-source-software/smartphone-ad-hoc-networking-span)).

O Wi-Fi Direct permite conexões individuais entre estações, embora, neste caso, um dos dispositivos realmente funcione como um access point virtual. O Wi-Fi Direct depende da configuração Wi-Fi protegida (Wi-Fi Protected Setup, WPS), que tem muitas vulnerabilidades. O Android aceita funcionar como um Wi-Fi Direct AP, mas o iOS usa uma estrutura de conectividade multipeer proprietária. No entanto, você pode conectar um dispositivo iOS a outro dispositivo executando um Wi-Fi Direct SoftAP.

Há também produtos em malha wireless de fornecedores como Netgear e Google que permitem que todos os tipos de dispositivos sem fio participem de uma rede ponto a ponto. Esses produtos podem não ser interoperáveis, embora muitos agora sejam compatíveis com o padrão EasyMesh ([wi-fi.org/discover-wi-fi/wi-fi-easymesh](https://www.wi-fi.org/discover-wi-fi/wi-fi-easymesh)).

## Tethering e access points

Um smartphone pode compartilhar sua conexão web com outro dispositivo, como um PC. Quando essa conexão é compartilhada por Wi-Fi com vários outros dispositivos, o smartphone é descrito como um access point (hotspot). A conexão é compartilhada ao conectar o smartphone a um computador via cabo USB ou com um único computador via Bluetooth, prática também conhecida como [[tethering]]. No entanto, o termo “Wi-Fi tethering” é bastante usado para se referir a um access point.

Esse tipo de funcionalidade normalmente seria desativado quando o dispositivo estivesse conectado a uma rede corporativa, pois poderia ser usado para contornar mecanismos de segurança, como políticas de prevenção de perda de dados ou filtragem de conteúdo web.
