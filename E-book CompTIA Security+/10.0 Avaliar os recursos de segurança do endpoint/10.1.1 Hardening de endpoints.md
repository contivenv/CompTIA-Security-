

## Segurança do sistema operacional

A segurança do sistema operacional abrange muitas práticas que visam proteger contra acessos não autorizados, violações de dados, infecções por malwares e outras ameaças à segurança. Há muitas considerações e requisitos ao proteger um sistema operacional, pois os sistemas operacionais são produtos de software complicados e potentes, que operam no núcleo de todos os sistemas de informação. Muitos conceitos de segurança se aplicam à segurança do sistema operacional, incluindo controles de acesso, mecanismos de autenticação, configurações seguras, segurança de aplicativos, codificação segura, gerenciamento de patches, proteção de endpoint, treinamento de conscientização de usuários e monitoramento.

Hardening (proteção) descreve a alteração de um sistema operacional ou aplicativo para que ele funcione com segurança. A necessidade de hardening deve levar em conta as necessidades funcionais e de usabilidade, pois o hardening muitas vezes pode afetar negativamente a forma como os aplicativos funcionam ou operam entre si.

Os parâmetros de práticas recomendadas desempenham um papel crítico no hardening de dispositivos: eles fornecem um conjunto padrão de diretrizes ou listas de conferência para configurar dispositivos com segurança. Esses parâmetros, geralmente desenvolvidos por especialistas ou organizações de segurança cibernética, são um ponto de partida para os administradores de sistemas garantirem que os dispositivos sejam configurados de acordo com os padrões de segurança do setor. Muitos dos requisitos podem ser aplicados automaticamente por meio de um modelo de parâmetros de configuração. O princípio central é a mínima funcionalidade: um sistema deve executar apenas os protocolos e serviços necessários para os usuários legítimos. Isso reduz a superfície de ataque em potencial.

- As interfaces fornecem uma conexão à rede, e algumas máquinas podem ter mais de uma interface. Por exemplo, pode haver interfaces com fio e wireless ou uma interface de modem. Algumas máquinas podem vir com uma placa de interface de rede de gerenciamento. Se alguma dessas interfaces não for necessária, ela deve ser explicitamente desativada, em vez de simplesmente deixada sem uso.
- Os serviços fornecem uma biblioteca de funções para diferentes tipos de aplicativos. Alguns serviços oferecem suporte a recursos locais do sistema operacional e aplicativos instalados. Outros dão suporte a conexões remotas de clientes para aplicativos de servidor. Os serviços não utilizados devem ser desativados.
- As portas de serviço de aplicativos permitem que o software cliente se conecte a aplicativos em uma rede. Elas devem ser desativadas ou bloqueadas em um firewall se o acesso remoto não for necessário. É importante lembrar que um servidor pode ser configurado com uma porta não padrão. Por exemplo, um servidor HTTP pode ser configurado para usar a porta 8080 em vez da 80. Por outro lado, um malware pode tentar enviar dados não padronizados por uma porta aberta. Um sistema de detecção de intrusão deve detectar dados da rede que não correspondem ao formato de protocolo esperado.
- O armazenamento persistente armazena os dados do usuário gerados por aplicativos, além de credenciais em cache. A criptografia de disco é essencial para a segurança dos dados. Unidades autocriptografadas podem ser usadas para que todos os dados em repouso sejam sempre armazenados com segurança.

Também é importante estabelecer um ciclo de manutenção para cada dispositivo e manter-se atualizado sobre novas ameaças de segurança e respostas para os produtos de software específicos que você está executando.

## Estações de trabalho

As estações de trabalho operam na linha de frente das atividades de uma organização e exigem considerações particulares em relação ao hardening de endpoints em comparação com outros dispositivos. Devido às tarefas variadas e aos inúmeros aplicativos associados ao uso da estação de trabalho, elas geralmente têm uma grande superfície de ataque. As práticas de hardening para minimizar essa superfície de ataque incluem a remoção de softwares desnecessários, a limitação de privilégios administrativos, o gerenciamento rigoroso da instalação e de atualizações de aplicativos e muitas outras mudanças. Além disso, como os funcionários operam estações de trabalho, estratégias de segurança focadas no usuário são essenciais, incluindo treinamento regular e atividades de conscientização para educar os usuários sobre ameaças como phishing e promover comportamentos seguros, como a adoção de senhas fortes, uso responsável da Internet e manuseio cuidadoso de dados confidenciais, entre outras práticas.

É essencial configurar a estação de trabalho para maximizar a segurança, como atualizações automáticas ativas, bloqueios de tela, firewalls, proteção de endpoints, detecção e prevenção de intrusão, maior registro em log, criptografia, monitoramento e muitas outras proteções. Além disso, a necessidade de proteger dispositivos periféricos, como portas USB, é especialmente crítica para estações de trabalho devido à sua utilização diversificada e alta exposição. Muitas vezes, isso é alcançado por meio do uso de recursos de software para a proteção de endpoints e da implementação de políticas rígidas de controle de dispositivos. Por fim, dadas as diversas funções e responsabilidades das estações de trabalho, a segmentação é crucial para restringir as comunicações e limitar o potencial de propagação de malware ou invasores pela rede.

## Configurações de linha de base e definições de registro

É comum possuir [[baselines de configuração]] separadas para clientes desktop, servidores de arquivos e impressão, servidores de DNS, servidores de aplicação, servidores de serviços de diretório e outros tipos de sistemas. No Windows, as definições de configurações são armazenadas no registro. Em uma rede de domínio do Windows, cada computador associado ao domínio recebe configurações de política de um ou mais objetos de política de grupo (GPOs). Essas configurações de política são aplicadas ao registro sempre que um computador é inicializado. Quando os hosts são gerenciados centralmente e executam apenas aplicativos e serviços autorizados, não há grandes motivos para alterar os valores de registro relevantes para a segurança. Os direitos de modificar o registro devem ser concedidos apenas para contas de usuário e serviço com base no privilégio mínimo necessário. É possível configurar um sistema de detecção de intrusão baseado em host para alertar sobre eventos de registro suspeitos.

Um relatório de desvio da linha de base consiste em testar a configuração real dos hosts para garantir que ela atende ao modelo de linha de base. Em redes Windows, a ferramenta Microsoft Baseline Security Analyzer (MBSA) foi muito usada para validar a configuração de segurança. O MBSA e outras ferramentas de relatórios da Microsoft foram substituídos pelo Kit de Ferramentas de Conformidade de Segurança([docs.microsoft.com/pt-bt/windows/security/threat-protection/security-compliance-toolkit-10](https://docs.microsoft.com/en-us/windows/security/threat-protection/security-compliance-toolkit-10)).
