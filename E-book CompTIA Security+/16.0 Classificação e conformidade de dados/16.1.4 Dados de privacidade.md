Os dados de privacidade são informações sensíveis ou de identificação pessoal associadas à identidade pessoal, financeira ou social de um indivíduo, incluindo dados que, se expostos ou tratados incorretamente, podem infringir os direitos de privacidade de um indivíduo. Exemplos de dados de privacidade incluem nomes, endereços, informações de contato, números de previdência social, prontuários médicos, transações financeiras e, geralmente, quaisquer outros dados que possam ser usados para identificar uma pessoa específica. Os dados de privacidade e os dados confidenciais têm certas semelhanças. Ambos exigem proteção devido à sua natureza sensível. O acesso não autorizado, a divulgação ou o uso indevido de dados de privacidade ou confidenciais podem afetar negativamente indivíduos ou organizações.

Além disso, tanto os dados de privacidade quanto os dados confidenciais estão sujeitos a considerações legais e éticas. As organizações devem cumprir as leis e os regulamentos relevantes, como as leis de proteção de dados e privacidade, para proteger ambos os tipos de dados. No entanto, também existem diferenças notáveis entre dados de privacidade e dados confidenciais.

Os dados confidenciais abrangem qualquer informação que exija proteção devido à sua natureza confidencial, independentemente de pertencer a um indivíduo. Os exemplos incluem segredos comerciais, propriedade intelectual, demonstrativos financeiros, algoritmos privados, códigos-fonte e outras informações não públicas. Os dados de privacidade, por outro lado, referem-se especificamente a informações que podem identificar um indivíduo ou afetar sua privacidade. No caso dos dados confidenciais, as principais preocupações são proteger as informações contra acesso, uso ou divulgação não autorizados para manter a competitividade dos negócios, proteger a propriedade intelectual ou preservar a integridade dos dados sensíveis da empresa.

Já no caso dos dados de privacidade, o foco está na proteção de informações pessoais para preservar os direitos de privacidade de um indivíduo, evitar o roubo de identidade e manter a confidencialidade dos dados pessoais. Os dados de privacidade estão intimamente associados aos direitos dos indivíduos de controlar o uso e a divulgação de suas informações pessoais. Os indivíduos têm o direito de acessar, corrigir e solicitar a exclusão de seus dados de privacidade. Em contrapartida, dados confidenciais normalmente não conferem direitos específicos aos [[titulares dos dados]], pois estão mais relacionados a informações proprietárias das organizações. O tratamento dos dados de privacidade geralmente requer o consentimento explícito do titular dos dados para sua coleta, uso e divulgação, particularmente em conformidade com as leis e os regulamentos de privacidade. Já os dados confidenciais, embora protegidos, podem não exigir necessariamente o consentimento individual para seu tratamento, pois estão associados a informações internas ou relacionadas a negócios.

A privacidade e os dados confidenciais compartilham semelhanças em termos de sensibilidade e considerações legais. No entanto, o escopo, o foco, os direitos do titular dos dados e os requisitos de consentimento são diferentes. Embora ambos os tipos de dados exijam tratamento e proteção criteriosos, os dados de privacidade estão relacionados explicitamente a informações pessoais e direitos de privacidade individuais.

## Implicações legais

A proteção dos dados de privacidade tem implicações legais locais, nacionais e globais significativas. Muitos países têm leis e regulamentos de privacidade específicos que determinam como os dados pessoais devem ser tratados dentro de sua jurisdição. Essas leis definem os direitos dos indivíduos, as responsabilidades das organizações e os procedimentos para proteção de dados e garantia de privacidade. A nível nacional, as autoridades de proteção de dados ou os órgãos de vigilância impõem as leis de privacidade e supervisionam a conformidade. Eles têm autoridade para investigar violações de dados, emitir multas e tomar medidas legais contra organizações que não protejam os dados de privacidade ou violem os direitos de privacidade dos indivíduos. O Regulamento Geral sobre a Proteção de Dados (RGPD) na União Europeia teve um impacto substancial em todo o mundo, estabelecendo altos padrões de privacidade e proteção de dados. Ele se aplica a organizações que processam os dados pessoais de residentes da UE, independentemente de sua localização física. Esse efeito extraterritorial garante que as organizações em todo o mundo cumpram os princípios do RGPD ao lidar com os dados pessoais dos cidadãos da UE. As transferências transfronteiriças de dados também estão sujeitas a requisitos e restrições específicos. Por exemplo, o RGPD restringe a transferência de dados pessoais para fora do Espaço Econômico Europeu, a menos que existam proteções adequadas para proteger os dados de privacidade. A compreensão e o cumprimento desses requisitos legais são essenciais para evitar consequências legais, manter a confiança dos indivíduos e promover uma cultura global de privacidade e proteção de dados.

## Funções e responsabilidades

Controlador de dados e processador de dados são duas funções distintas definidas pelos regulamentos de proteção de dados, como o RGPD. Embora ambos lidem com dados pessoais, essas funções têm semelhanças e diferenças importantes. O controlador de dados e o processador de dados estão envolvidos no tratamento de dados pessoais. Ambas as funções são responsáveis por garantir a proteção de dados pessoais em conformidade com as leis e os regulamentos de proteção de dados. O controlador de dados e o processador de dados também devem cumprir as leis de proteção de dados. Eles são obrigados a processar dados pessoais de forma legal, segura e transparente.

A principal distinção reside em suas funções e responsabilidades. O controlador de dados é a entidade ou organização que determina os propósitos e meios de processamento de dados pessoais. Eles têm controle geral e responsabilidade pelo processamento de dados pessoais. O controlador de dados decide por que e como os dados pessoais são processados. Ele exerce autoridade pela tomada de decisão, define os propósitos do processamento de dados e determina as categorias de dados a serem processados. Os controladores de dados têm obrigações e responsabilidades legais diretas de acordo com as leis de proteção de dados. Eles são responsáveis por lidar com a conformidade, obter o consentimento apropriado dos titulares dos dados, fornecer avisos de privacidade, implementar políticas e procedimentos de proteção de dados e lidar com solicitações de titulares de dados. O processador de dados é aquele que processa os dados pessoais em nome do controlador de dados, atuando sob a autoridade e instruções do controlador de dados. Os processadores de dados não têm poder de decisão independente sobre os dados pessoais. Eles processam dados apenas conforme instruído pelo controlador de dados. Os processadores de dados têm obrigações legais de processar dados pessoais apenas para os fins definidos pelo controlador de dados. Eles devem implementar medidas de segurança apropriadas, manter a confidencialidade e a integridade dos dados e cooperar com o controlador de dados para cumprir suas obrigações legais. Os processadores de dados também são obrigados a manter registros de suas atividades de processamento. Os processadores de dados incluem provedores de serviços em nuvem ou empresas de processamento de folha de pagamento, por exemplo.

O termo “titular de dados” refere-se a um indivíduo cujos dados pessoais são processados por uma organização ou outra entidade. Eles são os indivíduos a quem os dados pessoais se referem. Os titulares dos dados detêm certos direitos e proteções de acordo com as leis de proteção de dados, como a Lei Geral de Proteção de Dados (LGPD) e a Lei de privacidade do consumidor da Califórnia (California Consumer Privacy Act, CCPA). Um dos direitos concedidos aos titulares dos dados é o direito de acesso, o que significa que eles têm o direito de solicitar acesso aos seus dados pessoais e obter informações sobre como estão sendo processados. Os titulares podem questionar sobre os propósitos do processamento, as categorias dos dados que estão sendo processados, os destinatários dos dados e a duração da retenção dos dados. Os titulares dos dados têm o direito de retificação, o que significa que, se descobrirem que os dados pessoais mantidos por uma organização estão imprecisos ou incompletos, eles têm o direito de solicitar sua correção para garantir que seus dados pessoais estejam atualizados e precisos. Os titulares dos dados também têm o direito de solicitar a exclusão ou remoção de seus dados pessoais em determinadas circunstâncias.

Por exemplo, se os dados não forem mais necessários para os fins que justificaram sua coleta ou se o titular dos dados retirar seu consentimento para o processamento destes, ele poderá solicitar sua exclusão. Os titulares dos dados podem solicitar a restrição do processamento de seus dados pessoais. Isso implica que, embora os dados confidenciais ainda possam ser armazenados, eles não podem mais ser processados, exceto sob condições específicas. Esse direito dá aos titulares dos dados controle sobre o uso contínuo de seus dados pessoais. A portabilidade de dados é outro direito concedido aos titulares dos dados. Os titulares têm o direito de receber seus dados pessoais em um formato comumente usado e legível por máquina, o que garante sua capacidade de mover e transferir suas informações pessoais conforme desejado. Os titulares dos dados têm o direito de se opor ao processamento dos seus dados pessoais por motivos específicos. Por exemplo, quando um titular acredita que seus dados estão sendo processados para fins que não são legítimos ou deseja se opor às atividades de marketing direto. Por último, os titulares dos dados têm o direito de retirar seu consentimento para o processamento dos seus dados pessoais. Se o processamento for baseado em seu consentimento, eles poderão revogá-lo a qualquer momento, e a organização deverá interromper o processamento dos dados de maneira conforme.

Os titulares dos dados exercem esses direitos entrando em contato com o Controlador de dados, que garante que os direitos do titular dos dados sejam respeitados, facilita o exercício desses direitos e responde a eventuais preocupações ou solicitações dos titulares dos dados.

## Direito ao esquecimento

O “direito ao esquecimento” é um princípio fundamental descrito na Lei Geral de Proteção de Dados (LGPD) que concede aos titulares de dados o direito de solicitar a exclusão ou remoção dos seus dados pessoais sob certas circunstâncias. Isso permite que os indivíduos tenham suas informações pessoais removidas de plataformas online, bancos de dados ou outras fontes em que seus dados estejam sendo processados e disponibilizados publicamente.

O direito ao esquecimento reconhece a importância da privacidade individual e do controle sobre os dados pessoais. Ao receber uma solicitação de exclusão válida, o Controlador de dados deve apagar imediatamente os dados pessoais, a menos que haja motivos legítimos para recusar a solicitação. Esse direito se estende à remoção de dados dos sistemas da organização e a terceiros com os quais os dados tenham sido compartilhados ou disponibilizados publicamente. Esse direito pode ser limitado se o processamento de dados pessoais for necessário para o exercício do direito de liberdade de expressão e informação, o cumprimento de uma obrigação legal ou o estabelecimento, exercício ou defesa de reivindicações legais. O direito ao esquecimento funciona como um mecanismo para que os indivíduos recuperem o controle sobre suas informações pessoais. Ele promove a privacidade e a proteção de dados, permitindo que os titulares removam dados pessoais quando não for mais necessário ou lícito retê-los.

## Propriedade dos dados confidenciais

A questão da propriedade relacionada aos dados confidenciais é um tópico complexo. De maneira geral, não é fácil atribuir noções tradicionais de propriedade aos dados confidenciais. A propriedade dos dados confidenciais muitas vezes não é considerada nos termos do direitos de propriedade tradicionais. De acordo com muitas leis de proteção de dados, como a LGPD, a ênfase é colocada nos direitos e proteções do titular dos dados, em vez de determinar a propriedade. O titular dos dados tem controle sobre seus dados pessoais e pode exercer certos direitos, como o direito de acessar, retificar e excluir seus dados.

No entanto, as organizações que coletam e processam dados pessoais são consideradas guardiãs ou zeladoras dos dados, e não proprietárias. Elas têm responsabilidades legais e éticas de lidar com dados pessoais de forma lícita e segura e respeitar os direitos dos titulares dos dados. É importante observar que os dados confidenciais geralmente consistem em informações sobre indivíduos, e esses indivíduos têm um forte interesse em proteger suas informações pessoais. As leis de proteção de dados visam fornecer aos indivíduos o controle e proteção sobre seus dados pessoais, garantindo transparência, consentimento e práticas justas de processamento. Embora o conceito de propriedade possa não se aplicar diretamente aos dados confidenciais, os indivíduos têm direitos e controle sobre suas informações pessoais, e as organizações são legalmente responsáveis por lidar com os dados de forma responsável. O foco está em proteger os direitos de privacidade e garantir a proteção de dados, em vez de determinar a propriedade de forma tradicional.

## Inventários e retenção de dados

As leis de privacidade afetam profundamente os [[inventários de dados]] e as práticas de retenção de dados nas organizações. Essas leis, como a LGPD e a CCPA, exigem que as organizações mantenham um registro detalhado dos dados pessoais que coletam, processam e armazenam. Inventários de dados oferecem uma visão abrangente dos tipos de dados tratados, das finalidades do processamento, da base legal e dos destinatários dos dados, garantindo transparência e responsabilização, pois permitem que as organizações compreendam claramente e documentem suas atividades de tratamento de dados em conformidade com as leis de privacidade. As leis de privacidade estipulam que as organizações devem ter uma base legal para o processamento de dados pessoais. Os inventários de dados são essenciais para identificar os fundamentos legais para o processamento de dados. Ao documentar a base legal para cada categoria de dados pessoais, as organizações podem garantir que suas atividades de processamento estejam alinhadas aos propósitos legais especificados nas leis de privacidade. As organizações devem coletar e processar apenas os elementos necessários dos dados pessoais para fins específicos e legítimos. Os inventários de dados ajudam as organizações a avaliar os dados pessoais que coletam, garantindo que coletem apenas as informações necessárias. Ao manter o inventário atualizado, as organizações podem alinhar suas práticas aos princípios de minimização de dados e limitação de propósito.

A [[retenção de dados]] é outro aspecto impactado pelas leis de privacidade. As organizações devem reter os dados pessoais apenas pelo tempo necessário para cumprir o propósito pretendido ou conforme exigido por lei. Os inventários de dados ajudam as organizações a determinar os períodos de retenção apropriados para diferentes categorias de dados pessoais, garantindo a conformidade com os requisitos de limitação para o armazenamento de dados. Manter registros precisos no inventário de dados permite que as organizações excluam ou tornem os dados anônimos de forma segura quando não forem mais necessários. As leis de privacidade concedem aos indivíduos vários direitos, como o direito de acessar seus dados pessoais. Os inventários de dados são fundamentais para facilitar o exercício desses direitos. Ao manter inventários detalhados, as organizações podem responder prontamente às solicitações dos titulares dos dados, fornecer aos indivíduos acesso aos seus dados, corrigir imprecisões e atender aos pedidos de exclusão em conformidade com as leis de privacidade.

Além disso, as leis de privacidade exigem a implementação de medidas de segurança sólidas para proteger os dados pessoais. Os inventários de dados ajudam as organizações a identificar seus tipos de dados pessoais e todos os requisitos de segurança associados. Ao entender claramente os dados que processam, as organizações conseguem implementar medidas técnicas e organizacionais adequadas para proteger os dados pessoais contra acesso não autorizado, perda ou alteração, garantindo assim o cumprimento das obrigações de segurança conforme as leis de privacidade.
