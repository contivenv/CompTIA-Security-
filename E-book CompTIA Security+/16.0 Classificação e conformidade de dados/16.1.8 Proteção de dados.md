A classificação dos dados como “em repouso”, “em movimento” e “em uso” é essencial para medidas eficazes de proteção e segurança de dados. Ao analisar os dados com base em seu estado (em repouso, em movimento, em uso), as organizações conseguem adaptar medidas e controles de segurança para lidar com os riscos e requisitos específicos associados a cada estado de dados. Essa classificação ajuda as organizações a identificar vulnerabilidades, priorizar investimentos em segurança e garantir proteções adequadas para proteger dados confidenciais durante todo o seu ciclo de vida. Isso também facilita a conformidade com os regulamentos de proteção de dados e as práticas recomendadas da indústria.

- **Dados em repouso**: nesse estado os dados estão em algum tipo de mídia de armazenamento persistente. Exemplos de tipos de dados que podem estar em repouso são informações financeiras armazenadas em bancos de dados, mídia audiovisual arquivada, políticas operacionais e outros documentos administrativos, dados de configuração do sistema e outros. Nesse estado, geralmente é possível criptografar os dados por meio de técnicas como criptografia de disco completo, criptografia de banco de [[dados e criptografia]] no âmbito do arquivo ou da pasta. Também é possível aplicar permissões — listas de controle de acesso (Access Control Lists, ACL) — para garantir que apenas usuários autorizados possam ler ou modificar os dados. As ACLs só podem ser aplicadas se o acesso aos dados for totalmente mediado por meio de um sistema operacional confiável.
- **Dados em trânsito (ou dados em movimento)**: nesse estado os dados são transmitidos por uma rede. Exemplos de tipos de dados que podem estar em trânsito são tráfego de sites, tráfego de acesso remoto, dados sendo sincronizados entre repositórios de nuvem e outros. Nesse estado, os dados podem estar protegidos por um protocolo de criptografia de transporte, como TLS ou IPSec.

Nota:

Com os dados em repouso, há um desafio de criptografia maior do que com os dados em trânsito, pois as chaves de criptografia precisam ser mantidas seguras por mais tempo. A criptografia de transporte pode usar chaves efêmeras (de sessão).

- **Dados em uso (ou dados em processamento)**: nesse estado os dados estão presentes na memória volátil, como registros e cache de RAM ou CPU do sistema. Exemplos de tipos de dados que podem estar em uso são documentos abertos em um aplicativo de processamento de texto, dados de banco de dados que estão sendo modificados no momento, registros de eventos sendo gerados enquanto um sistema operacional está em execução e outros. Quando um usuário manipula dados, esses dados normalmente precisam ser descriptografados ao passarem do estado em repouso para o estado em uso. Os dados podem permanecer descriptografados por uma sessão de trabalho inteira, o que os coloca em risco. Entretanto, mecanismos de trusted execution environment (TEE), como o Intel Software Guard Extensions, conseguem criptografar dados enquanto estão na memória, impedindo que um processo não confiável consiga decodificar as informações.

## Métodos de proteção de dados

 
|Método|Descrição|
|---|---|
|Restrições geográficas|As restrições geográficas envolvem a limitação do acesso aos dados com base em localizações geográficas específicas. Isso garante que os dados sejam acessíveis apenas de regiões aprovadas, o que proporciona controle e segurança adicionais. Um caso de uso comum das restrições geográficas envolve os serviços de computação em nuvem e armazenamento de dados. Ao utilizar plataformas de nuvem para armazenar seus dados, as organizações ou datacenters externos podem precisar impor restrições geográficas para especificar onde seus dados podem ser armazenados e processados de modo a cumprir as leis e regulamentos de proteção de dados.|
|Encryption (criptografia)|A criptografia converte os dados em um formato codificado que só pode ser acessado ou decifrado com uma chave de criptografia ou senha. Isso protege a confidencialidade dos dados e garante que, mesmo se os dados forem interceptados, permaneçam ilegíveis para partes não autorizadas.|
|Hashing|O hash envolve a conversão de dados em uma cadeia de caracteres de tamanho fixo usando um algoritmo de hash. O hash é comumente usado para verificar a integridade dos dados e armazenar senhas com segurança.|
|Mascaramento|O mascaramento envolve a substituição de dados confidenciais por valores fictícios ou parcialmente ocultos, o que preserva o formato e o tamanho dos dados originais. Isso evita a exposição de informações confidenciais e geralmente é usado para ocultar campos de dados confidenciais e caracteres de senha inseridos em formulários.|
|Tokenização|A tokenização substitui dados confidenciais por um token gerado aleatoriamente, enquanto armazena com segurança os dados originais em um local separado. Os tokens não têm valor significativo, o que reduz o risco de acesso não autorizado ou a exposição de informações confidenciais. Um caso de uso comum da tokenização de dados é em sistemas de processamento de pagamentos. Quando os clientes fazem um pagamento, suas informações confidenciais do cartão de pagamento, como números do cartão de crédito, são substituídas por um token gerado aleatoriamente. Esse token é então usado para representar os dados do cartão de pagamento durante as transações e é armazenado no banco de dados do sistema.|
|Ofuscação|A ofuscação envolve modificar dados para torná-los difíceis de entender ou fazer engenharia reversa sem alterar a funcionalidade. O desenvolvimento de softwares geralmente usa técnicas de ofuscação para proteger a propriedade intelectual do código-fonte e impedir o acesso não autorizado a detalhes essenciais. Exemplos de ofuscação são mascaramento de dados, conversão de tipo de dados e hash.|
|Segmentação|A segmentação é um método de proteger dados dividindo redes, dados e aplicativos em componentes isolados de forma a melhorar a proteção de dados confidenciais, limitar o impacto de uma violação e melhorar a segurança da rede. A segmentação ajuda a restringir o acesso com base nas funções do usuário, privilégios, localização ou outros critérios. Isso ajuda a limitar a exposição, concedendo acesso apenas aos segmentos de dados específicos necessários para usuários ou processos autorizados. Um caso de uso comum da segmentação de dados é em sistemas ou registros eletrônicos de saúde (Electronic Health Records, EHR). Os dados do paciente geralmente são categorizados e segmentados nesses sistemas com base em vários fatores, como condições médicas, departamentos ou níveis de acesso. A segmentação de dados permite que os profissionais de saúde controlem e limitem o acesso a informações confidenciais do paciente com base no princípio do privilégio mínimo. Os diferentes prestadores de serviços de saúde, especialistas ou departamentos só poderão ter acesso aos dados específicos do paciente que forem relevantes para o exercício de suas funções ou responsabilidades de tratamento.|
|Restrições de permissão|As restrições de permissão envolvem o controle do acesso aos dados com base nas permissões do usuário. Elas asseguram que somente as funções ou os indivíduos autorizados possam visualizar, modificar ou interagir com elementos de dados específicos, o que reduz o risco de acesso não autorizado, violações de dados ou uso indevido acidental. As listas de controle de acesso, o controle de acesso baseado em função, o controle de acesso baseado em regras, o controle de acesso obrigatório, o controle de acesso baseado em atributos e outros métodos aplicam o princípio do privilégio mínimo.|
