Para aplicar políticas e procedimentos de tutela de dados, organizações menores podem classificar e digitar dados manualmente. No entanto, uma organização que cria e coleta grandes quantidades de dados pessoais geralmente precisa usar ferramentas automatizadas para ajudar nessa tarefa. Também pode ser preciso proteger dados valiosos de propriedade intelectual. Os produtos de prevenção contra perda de dados (DLP) tanto automatizam a descoberta e classificação dos tipos de dados como aplicam regras para que os dados não sejam visualizados ou transferidos sem a devida autorização. Essas soluções geralmente consistem nos seguintes componentes:

- **Servidor de políticas**: para configurar regras e políticas de classificação, confidencialidade e privacidade, registrar incidentes e compilar relatórios.
- **Agentesde endpoint**: para aplicar políticas nos computadores dos clientes, mesmo quando não estão conectados à rede.
- **Agentes de rede**: para inspecionar as comunicações nas bordas da rede e interagir com servidores web e de mensagens para aplicar políticas.

Os agentes de DLP verificam o conteúdo em formatos estruturados, como um banco de dados com um modelo formal de controle de acesso, ou em formatos não estruturados, como email ou documentos de processamento de texto. A transformação de dados é aplicada a dados não estruturados para torná-los em um formato consistente e passível de varredura, facilitando a aplicação de políticas. A transferência de conteúdo para mídias removíveis, como dispositivos USB, ou por email, mensagens instantâneas ou até mesmo redes sociais, pode ser bloqueada se não estiver em conformidade com uma política predefinida. A maioria das soluções de DLP pode estender os mecanismos de proteção aos serviços de armazenamento em nuvem usando um proxy para mediar o acesso ou a API do provedor de serviços em nuvem para executar a verificação e a aplicação de políticas.

A remediação é a ação que o software de DLP realiza quando detecta uma violação de política. Estes são mecanismos de remediação comuns:

- **Somente alerta**: a cópia é permitida, mas o sistema de gestão a registra como um incidente e pode alertar um administrador.
- **Bloqueio**: o usuário é impedido de copiar o arquivo original, mas mantém o acesso a ele. O usuário pode ou não ser alertado sobre a violação da política, mas o mecanismo de gestão a registrará como um incidente.
- **Quarentena**: o acesso ao arquivo original é negado ao usuário (ou possivelmente a qualquer usuário). Para isso, o arquivo pode ser criptografado ou movido para uma área de quarentena no sistema de arquivos.
- **Tombstone (marco de exclusão)**: o arquivo original é colocado em quarentena e substituído por outro que descreve a violação da política e como o usuário pode liberá-lo novamente.

Quando configurada para proteger um canal de comunicação, como o email, a remediação de DLP pode utilizar mecanismos do lado do cliente ou do lado do servidor. Por exemplo, algumas soluções de DLP impedem a anexação de arquivos ao email antes do envio. Outras verificam os anexos do email e o conteúdo da mensagem e removem determinados dados ou impedem que o email chegue ao destinatário.
