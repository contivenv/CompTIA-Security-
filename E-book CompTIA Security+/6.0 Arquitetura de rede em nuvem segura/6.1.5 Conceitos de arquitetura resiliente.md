# 

Um dos benefícios da nuvem é o potencial de fornecer serviços resilientes a falhas em diferentes níveis, como componentes, servidores, redes locais, sites, datacenters e redes de longa distância (WAN). O provedor de serviços em nuvem (CSP) utiliza uma camada de [[virtualização]] para garantir que os recursos de computação, armazenamento e rede atendam aos critérios de disponibilidade definidos no seu SLA.

No que diz respeito aos níveis de desempenho de armazenamento, [[alta disponibilidade (HA)]] refere-se ao armazenamento provisionado com garantia de 99,99% de tempo de atividade ou superior. Assim como acontece com a arquitetura local, o CSP usa redundância para disponibilizar vários controladores de disco e dispositivos de armazenamento para um pool de recursos de armazenamento. Os dados podem ser replicados entre pools ou grupos, com cada pool suportado por recursos de hardware separados.

## Replicação

A [[replicação]] de dados permite que as empresas copiem dados para os locais onde eles podem ser utilizados de forma mais eficaz. A nuvem pode ser usada como uma área de armazenamento central, disponibilizando dados a todas as unidades de negócios. A replicação de dados requer conexões de rede de baixa latência, segurança e integridade de dados. Os CSPs oferecem várias modalidades de desempenho de armazenamento de dados ([cloud.google.com/storage/docs/storage-classes](https://cloud.google.com/storage/docs/storage-classes)). Os termos “hot storage” (armazenamento a quente) e “cold storage” (armazenamento a frio) referem-se à rapidez com que os dados são recuperados. O armazenamento a quente recupera dados mais rapidamente do que o armazenamento a frio, porém quanto mais rápida a recuperação, maior o custo.

Diferentes aplicativos têm necessidades de replicação próprias. Um banco de dados geralmente precisa de replicação síncrona de baixa latência, pois uma transação em geral não pode ser considerada completa até que tenha sido feita em todas as réplicas. Um mecanismo para replicar arquivos de dados no armazenamento de backup pode não ter requisitos tão altos, dependendo da criticidade dos dados.

## Alta disponibilidade em todas as zonas

Os CSPs dividem o mundo em regiões. Cada região é independente das outras. As regiões são divididas em zonas de disponibilidade, que têm datacenters independentes com sua própria energia, resfriamento e conectividade de rede. Você pode optar por hospedar dados, serviços e instâncias de VM em uma região específica para fornecer um serviço de latência mais baixa aos clientes. O provisionamento de recursos em várias zonas e regiões também pode melhorar o desempenho e aumentar a redundância, mas requer um nível adequado de desempenho de replicação.

Consequentemente, os CSPs oferecem várias modalidades de replicação, que representam diferentes níveis de serviço de alta disponibilidade:

- **Replicação local**: replica seus dados dentro de um único datacenter na região em que a conta de armazenamento foi criada. As réplicas estão frequentemente em domínios de falha e domínios de upgrade separados.
- **Replicação regional** (também chamada de armazenamento com redundância de zona): replica seus dados entre vários datacenters dentro de uma ou duas regiões. Isso protege os dados e o acesso no caso de um único datacenter ser destruído ou ficar off-line.
- **Armazenamento com redundância geográfica (GRS)**: replica seus dados para uma região secundária distante da região primária. Isso protege os dados no caso de uma interrupção ou desastre regional.
