

A [[virtualização de aplicativos]] é um tipo mais limitado de infraestrutura de desktop virtual (VDI). Em vez de executar toda a área de trabalho do cliente como uma plataforma virtual, o cliente acessa um aplicativo hospedado em um servidor ou transmite o aplicativo do servidor para o cliente para processamento local. A maioria das soluções de virtualização de aplicativos é baseada no Citrix XenApp (antigo MetaFrame Presentation Server), embora a Microsoft tenha desenvolvido um produto App-V com sua linha Windows Server e a VMware tenha o produto ThinApp. Esses tipos de solução são frequentemente usados com aplicativos de área de trabalho remota HTML5, chamados de soluções “sem cliente” porque os usuários podem acessá-los por meio de um software comum de navegador web.

A [[conteinerização]] é uma tecnologia poderosa que transformou o empacotamento e a implantação de aplicativos. Os contêineres encapsulam todos os componentes necessários para o software, incluindo código, bibliotecas e configurações, dentro de uma unidade portátil denominada "contêiner". Isolar o software dessa forma garante um comportamento consistente do aplicativo, independentemente da plataforma subjacente em que ele é executado. Contêineres de software são análogos ao uso de contêineres físicos na indústria de transporte marítimo.

Por exemplo, todos os componentes necessários para fornecer um aplicativo web podem ser implantados em contêineres e gerenciados por meio de uma plataforma de contêineres, como o Docker (Docker representa o navio cargueiro que transporta os contêineres). Isso difere significativamente da instalação dos componentes a partir de um repositório de código diretamente "dentro" do sistema operacional e da edição manual de todos os arquivos de configuração associados. Ao utilizar contêineres, diferentes componentes do aplicativo podem ser "trocados", evitando muitas das complexidades associadas à instalação e configuração tradicional de softwares. Métodos tradicionais frequentemente enfrentam problemas porque a configuração do sistema de destino possui versões diferentes de softwares e bibliotecas das exigidas pelos novos componentes. O chamado "pesadelo de dependências" normalmente se manifesta como um ciclo frustrante de tentativas de resolver conflitos ao atualizar ou fazer downgrade de dependências, resultando em novos problemas ou na quebra de funcionalidades já existentes. Isso desperdiça tempo e esforço dos desenvolvedores que precisam navegar por um verdadeiro labirinto de dependências. Aplicações conteinerizadas evitam isso porque são unidades autocontidas, cada uma contendo uma cópia independente de suas dependências.

Os hipervisores desempenham um papel fundamental na virtualização ao gerenciar múltiplas máquinas virtuais (VMs) em uma única plataforma de hardware. Existem dois tipos principais de hipervisores: Tipo 1 e Tipo 2. Os hipervisores do Tipo 1, também conhecidos como hipervisores bare-metal, são executados diretamente no hardware físico, oferecendo alto desempenho e eficiência, o que os torna ideais para ambientes corporativos. Exemplos incluem VMware ESXi e Microsoft Hyper-V. Os hipervisores do Tipo 2, ou hipervisores hospedados, são executados sobre um sistema operacional hospedeiro e geralmente são usados para desenvolvimento e testes. Exemplos de hipervisores Tipo 2 incluem VMware Workstation e Oracle VirtualBox.
