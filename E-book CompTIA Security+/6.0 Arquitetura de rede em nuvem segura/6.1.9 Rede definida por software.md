# 

A IaC é parcialmente facilitada por dispositivos de rede físicos e virtuais que são totalmente configuráveis por meio de scripts e APIs. À medida que as redes se tornam mais complexas, envolvendo milhares de computadores e dispositivos físicos e virtuais, fica mais difícil implementar políticas de rede, como garantir a segurança e gerenciar o fluxo de tráfego.

Com tantos dispositivos para configurar, é melhor dar um passo para trás e considerar um modelo abstrato de como a rede funciona. Neste modelo, as funções de rede podem ser divididas em três “planos”:

- **Plano de gerenciamento**: monitora as condições do tráfego e o status da rede.
- **Plano de controle**: toma decisões sobre como o tráfego deve ser priorizado, protegido e para onde deve ser encaminhado.
- **Plano de dados**: gerencia a comutação e o roteamento do tráfego e impõe controles de acesso de segurança.

Uma [[aplicação de rede definida por software (SDN)]] pode ser usada para definir decisões de política no plano de controle. Essas decisões são então implementadas no plano de dados por um aplicativo controlador de rede, que faz interface com os dispositivos de rede por meio de APIs. A interface entre os aplicativos SDN e o controlador SDN é descrita como a API “northbound”, enquanto a interface entre o controlador e os dispositivos é a API “southbound”. A SDN pode ser usada para gerenciar dispositivos físicos compatíveis, mas também switches virtuais, roteadores e firewalls. A arquitetura que possibilita a implantação rápida de redes virtuais utilizando máquinas virtuais (VMs) de uso geral e contêineres é chamada de [[virtualização de funções de rede (NFV)]] ([redhat.com/en/topics/virtualization/what-is-nfv](https://www.redhat.com/en/topics/virtualization/what-is-nfv)).

Graças a essa arquitetura, os administradores de rede e segurança não têm o trabalho e a complexidade de definir as configurações do dispositivo corretamente para impor uma política desejada. Ela também permite a implantação (ou provisionamento) totalmente automatizada de links de rede, dispositivos e servidores. Isto torna a SDN uma parte importante das mais recentes tecnologias de automação e orquestração.
