Pode-se chamar de [[backdoor]] qualquer tipo de método de acesso a um host que dribla o método de autenticação padrão e fornece controle administrativo ao usuário remoto. Um [[Trojan de acesso remoto (RAT)]] é um malware de backdoor que imita a funcionalidade de programas legítimos de controle remoto, mas é projetado especificamente para operar de forma oculta. Depois que o RAT é instalado, ele permite que o autor da invasão acesse o host, envie arquivos e instale software ou use técnicas de LoTL para causar ainda mais danos.

Nota:

Nesse contexto, a sigla RAT também pode designar uma ferramenta de administração remota. Um host sob o controle alheio mal-intencionado é por vezes descrito como um zumbi.

Um host comprometido pode ser instalado com um ou mais bots. Um bot é um script ou ferramenta automatizados que executa algum tipo de atividade mal-intencionada. Um grupo de bots sob o controle da mesma instância de malware pode ser manipulado como uma botnet pelo programa controlador. Uma [[botnet]] pode ser usada para diversos propósitos mal-intencionados, incluindo o desencadeamento de ataques distribuídos de negação de serviço (DDoS), campanhas de spam ou mineração de criptomoedas.

Descrição

A tela exibe um botão expansível chamado connection, selecionado para listar os hiperlinks para ip scanner, get pc info, get home info, server options (selecionado) e ip notify. Os botões à direita são: change port, set default port, set password, remove password, disconnect victim, restart server, remove server, close server, update server from local file e update server from URL file. Outros botões expansíveis no canto inferior esquerdo são: keys or messages, advanced, miscellaneous, fun manager, extra fun e local options.

Seja um backdoor utilizado como mecanismo de intrusão independente ou para gerenciar bots, o agente de ameaça precisa estabelecer uma conexão do host comprometido para um host ou rede de [[comando e controle (C2 ou C&C)]]. Essa conexão de rede geralmente é a melhor maneira de identificar a presença de um RAT, backdoor ou bot. Existem muitos meios de implementar uma rede de C&C como um [[canal secreto]] para evitar a detecção e filtragem. Historicamente, o protocolo de [[bate-papo via internet (IRC)]] foi muito popular. Já os métodos modernos provavelmente usam sequências de comandos incorporadas no tráfego HTTPS ou DNS.

Nota:

As backdoors podem ser criadas de outras maneiras além da infecção por malware. Os programadores podem criar backdoors em aplicações de software para fins de teste e desenvolvimento que, posteriormente, não são removidas quando o aplicativo é implantado. As backdoors também são criadas quando há uma configuração incorreta de software ou hardware que permite acesso a usuários não autorizados.
