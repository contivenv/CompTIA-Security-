[[Directory traversal (diretório transversal)]] é outro tipo de ataque de injeção realizado contra um servidor Web. O invasor solicita um arquivo fora do diretório root do servidor Web informando um caminho para acessar o diretório principal (`../`) Esse ataque pode ser bem-sucedido se a entrada não for filtrada corretamente, e as permissões de acesso ao arquivo permitirem que o processo do servidor Web o leia, grave ou execute.

O invasor pode usar um [[ataque de canonicalization (canonização)]] para disfarçar a natureza da entrada maliciosa. A canonização refere-se à maneira como o servidor converte entre os diferentes métodos pelos quais um recurso pode ser representado e submetido ao método mais simples (ou canônico) usado pelo servidor para processar a entrada. Exemplos de esquemas de codificação incluem entidades HTML e codificação percentual de conjunto de caracteres. Um invasor pode conseguir explorar vulnerabilidades no processo de canonização para executar a injeção de código ou facilitar o diretório transversal. Por exemplo, para executar um ataque de diretório transversal, o invasor pode enviar um URL como o seguinte:

`http://victim.foo/?show=../../../../etc/config`

Uma rotina de validação de entrada limitada impediria o uso da string ../ e recusaria a solicitação. Se o invasor enviou o URL usando a versão codificada dos caracteres, ele pode conseguir burlar a rotina de validação:

`http://victim.foo/?show=%2e%2e%2f%2e%2e%2f%2e%2e%2f%2e%2e%2fetc/config`

Um [[ataque de injeção]] de comando tenta fazer com que o servidor execute comandos do shell do sistema operacional e devolva o resultado ao navegador. Assim como no diretório transversal, o servidor Web normalmente deve ser capaz de impedir que os comandos operem fora diretório root do servidor e impedir que os comandos sejam executados com qualquer nível de privilégio que não seja o usuário “convidado” do servidor Web (que normalmente recebe apenas privilégios muito restritos). Um ataque de injeção de comando bem-sucedido encontraria alguma maneira de contornar essa segurança ou explorar um servidor Web que não está configurado corretamente.
