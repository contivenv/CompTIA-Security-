Normalmente, os servidores Web são configurados para registrar o tráfego HTTP que encontra um erro ou tráfego que corresponde a algum conjunto de regras predefinido. Isso pode preservar os indicadores de ataques tentativos e bem-sucedidos de repetição, falsificação e injeção.

O código de status de uma resposta pode revelar bastante sobre a solicitação e o comportamento do servidor. Códigos na faixa de 400 indicam erros baseados no cliente, enquanto códigos na faixa de 500 indicam erros baseados no servidor. Por exemplo, respostas repetidas 403 “Forbidden” (proibido) podem indicar que o servidor está rejeitando as tentativas de um cliente de acessar recursos aos quais não está autorizado. Uma resposta 502 “Bad Gateway” (Gateway incorreto) pode indicar que as comunicações entre o servidor de destino e seu servidor upstream estão sendo bloqueadas ou que o servidor upstream está inativo.

Além dos códigos de status, alguns softwares de servidor Web também registram informações de cabeçalho HTTP para solicitações e respostas. Isso pode fornecer uma imagem detalhada da composição de cada solicitação ou resposta, como informações sobre cookies.
