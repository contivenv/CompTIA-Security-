

Um sistema de controle de acesso garante que um sistema de informação atenda aos objetivos da tríade CIA. O controle de acesso rege como sujeitos/entidades podem interagir com objetos. As entidades são pessoas, dispositivos, processos de software ou qualquer outro sistema que possa solicitar e receber acesso a um recurso. Os objetos são os recursos. Um objeto pode ser uma rede, servidor, banco de dados, aplicativo ou arquivo. As entidades recebem direitos ou permissões relacionados a recursos.

O controle de acesso moderno costuma ser implementado como um sistema de [[gerenciamento de identidades e acessos]] (IAM). O IAM compreende quatro processos principais:

- [[Identificação]]: criar uma conta ou ID que represente exclusivamente o usuário, dispositivo ou processo na rede.
- [[Autenticação]]: comprova que uma entidade é quem ou o que afirma ser quando tenta acessar o recurso.Um fator de autenticação determina que tipo de credencial a entidade pode usar. Por exemplo, pessoas podem ser autenticadas por meio de uma senha, já um sistema de computador pode ser autenticado usando um token, como um certificado digital.
- [[Autorização]]: determina quais direitos as entidades devem ter sobre cada recurso e impõe esses direitos. Um modelo de autorização determina como esses direitos são concedidos. Por exemplo, em um modelo discricionário, o proprietário do objeto pode alocar direitos. Em um modelo obrigatório, os direitos são predeterminados por regras aplicadas pelo sistema e não podem ser alterados por nenhum usuário dentro do sistema.
- [[Contabilização]]: Monitora o uso autorizado de um recurso ou o uso de direitos por uma entidade e alerta quando é detectado ou tentado o uso não autorizado.

Descrição

Os processos são os seguintes: 1. Identificação: o responsável pelo sistema confirma a identidade do usuário e cria uma conta para representá-lo. 2. Autenticação: o usuário se autentica inserindo a credencial, que é comparada com a credencial armazenada em formato de hash. 3. Autorização: para cada ação realizada pela conta, uma lista de permissões é verificada para permitir ou negar a ação. As ações podem incluir leitura ou gravação; a lista verificada é a Access Control List (ACL). 4. Contabilização: o sistema registra o uso das permissões em um log. O usuário não pode impedir essa auditoria.

Nota:

Os servidores e protocolos que implementam essas funções também podem ser chamados de autenticação, autorização e contabilização (AAA). O uso do IAM para descrever fluxos de trabalho de segurança corporativa está se tornando mais predominante à medida que a importância do processo de identificação é melhor reconhecida.

Por exemplo, se você está configurando um site de comércio eletrônico e quer cadastrar usuários, precisa selecionar os controles apropriados para executar cada função:

- **Identificação**: garantir que os clientes sejam legítimos. Por exemplo, talvez você precise garantir que os endereços de cobrança e entrega sejam iguais e que não estejam tentando usar formas de pagamento fraudulentas.
- **Autenticação**: garantir que os clientes tenham contas exclusivas e que apenas eles possam gerenciar seus pedidos e informações de cobrança.
- **Autorização**: regras para garantir que os clientes possam fazer pedidos apenas quando possuírem mecanismos de pagamento válidos. Você pode operar programas de fidelidade ou promoções que autorizem determinados clientes a visualizar ofertas ou conteúdos exclusivos.
- **Contabilização**: o sistema deve registrar as ações de um cliente (para garantir que não possa negar que fez um pedido, por exemplo).

Nota:

Lembre-se de que esses processos se aplicam tanto às pessoas quanto aos sistemas. Por exemplo, você precisa garantir que seu servidor de comércio eletrônico possa autenticar sua identidade quando os clientes se conectarem a ele usando um navegador web.
