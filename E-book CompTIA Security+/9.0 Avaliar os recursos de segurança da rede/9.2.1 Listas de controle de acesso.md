

Uma lista de controle de acesso (Access Control List, ACL) enumera as permissões associadas a um dispositivo de rede que controla o tráfego ao nível da interface de rede, como um roteador ou um switch. As ACLs geralmente usam informações de pacote, como endereços IP de origem e destino, números de porta e o protocolo para decidir se devem permitir ou negar o tráfego. Elas são geralmente implementadas em dispositivos de rede para fornecer controle de tráfego em toda a rede, adicionando uma camada de segurança e eficiência. Em contraste, uma regra de firewall determina como os firewalls devem lidar com o tráfego que entra ou sai da rede quanto a endereços IP, intervalos de IP ou interfaces de rede específicos. Os firewalls geralmente fornecem controle da rede e dos aplicativos. Eles são criados para proteger um perímetro de rede, impedindo o acesso não autorizado à rede ou a partir dela. As regras de firewall podem se basear em vários fatores, como endereços IP, números de porta, protocolos ou até mesmo padrões de tráfego de aplicativos específicos.

As regras na ACL de um firewall são processadas de cima para baixo. Se o tráfego cumprir uma das regras, ele poderá passar; consequentemente, as regras mais específicas são colocadas no topo. A regra padrão final normalmente é bloquear qualquer tráfego que não tenha correspondido a uma regra ([[negação implícita]]). Se o firewall não tiver uma regra de negação implícita padrão, uma regra de negação explícita poderá ser adicionada manualmente ao final da ACL.

Cada regra pode especificar se o tráfego deve ser bloqueado ou permitido com base em vários parâmetros, muitas vezes chamados de tuplas. Se imaginarmos que cada regra é como uma linha em um banco de dados, então as tuplas seriam as colunas. Por exemplo, na captura de tela anterior, as tuplas incluem Protocolo, Origem (endereço), (Origem) Porta, Destino (endereço), (Destino) Porta e assim por diante.

Até mesmo o firewall de filtragem de pacotes mais simples pode ser difícil de configurar de forma segura. É essencial criar uma política por escrito que descreva o que um conjunto de regras de filtragem deve fazer e testar a configuração o máximo possível para garantir que as ACLs configuradas funcionem conforme o esperado. Além disso, deve-se testar e documentar as alterações feitas nas ACLs. Alguns outros princípios básicos são:

- Bloquear solicitações recebidas de endereços IP internos ou privados (que foram claramente falsificados).
- Bloquear solicitações recebidas de protocolos que deveriam funcionar apenas na rede local, como ICMP, DHCP ou tráfego de protocolo de roteamento.
- Usar o teste de penetração para confirmar se a configuração é segura. Registrar as tentativas de acesso e monitorar os logs em busca de atividades suspeitas.
- Seguir as etapas usuais para proteger o hardware no qual o firewall está sendo executado e usar a interface de gerenciamento.

Por exemplo, uma regra de firewall pode ser criada especificamente para permitir ou negar o tráfego com base nos números de porta TCP ou UDP em que um serviço opera. Se um servidor web em uma rede permitir apenas o tráfego HTTP e HTTPS recebido, as regras poderão ser configuradas para permitir o tráfego apenas nas portas 80 (HTTP) e 443 (HTTPS), as portas padrão para estes serviços. Da mesma forma, as regras podem ser definidas para restringir a entrada na rede de certos protocolos, como FTP ou SSH, se não forem necessários, reduzindo assim a possível superfície de ataque.

Além disso, você pode usar regras de firewall para restringir o tráfego de saída e evitar certos tipos de comunicação de dentro da rede. Por exemplo, uma regra pode bloquear todo o tráfego de saída para a porta 25 (SMTP) de modo a evitar que uma máquina comprometida na rede envie emails de spam.

## Sub-rede filtrada

Uma sub-rede filtrada, também conhecida como rede perimetral, cria uma camada adicional de proteção entre a rede interna de uma organização e a Internet. Uma sub-rede filtrada atua como uma zona neutra, separando servidores expostos publicamente de recursos confidenciais da rede interna para reduzir a exposição dos recursos internos a ameaças externas. Em termos práticos, a [[sub-rede filtrada]] geralmente hospeda serviços web, email, DNS ou FTP. Normalmente, esses sistemas devem ser acessíveis a partir da Internet pública, mas isolados de sistemas internos confidenciais para limitar o impacto de uma violação de um desses serviços. Ao colocar esses servidores na sub-rede filtrada, uma organização pode limitar os danos se esses servidores forem comprometidos.

Os firewalls são normalmente usados para criar e controlar o tráfego de ida e volta da sub-rede filtrada. O primeiro firewall, entre a Internet e a sub-rede filtrada, é configurado para permitir o tráfego para os serviços hospedados na sub-rede filtrada. O segundo firewall, entre a sub-rede filtrada e a rede interna, é configurado para bloquear a maioria (praticamente todo) do tráfego da sub-rede filtrada para a rede interna. Uma sub-rede filtrada é parte fundamental da arquitetura de segurança de uma rede e um exemplo importante de segmentação de rede como um tipo de controle de segurança.
