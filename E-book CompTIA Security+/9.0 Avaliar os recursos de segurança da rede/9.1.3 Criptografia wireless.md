# Security+ CertMaster Study (Versão em português) 7.0

Além do projeto do local, uma rede sem fio deve ser configurada com definições de segurança. Sem criptografia, qualquer pessoa pode interceptar e ler os pacotes que passam pela rede sem fio, desde que estejam dentro do seu alcance. As escolhas de segurança são determinadas pelo suporte do dispositivo aos vários padrões de segurança de Wi-Fi, pelo tipo de infraestrutura de autenticação e pela finalidade da WLAN. O padrão de segurança determina quais protocolos criptográficos são suportados, os meios de gerar a chave de criptografia e os métodos disponíveis para autenticar estações wireless quando tentam ingressar (ou se associar) à rede.

A primeira versão do [[Wi-Fi Protected Access (WPA)]] foi criada para corrigir vulnerabilidades críticas presentes no padrão anterior de [[Wired Equivalent Privacy (WEP)]] . Assim como o WEP, a versão 1 do WPA utiliza o cifrador de fluxo RC4, mas adiciona um mecanismo chamado [[Temporal Key Integrity Protocol (TKIP)]] para reforçar sua segurança.

## Configuração Wi-Fi protegida (WPS)

Como a configuração segura de um ponto de acesso é relativamente complexa para consumidores residenciais, os fabricantes desenvolveram um sistema para automatizar esse processo chamado [[Wi-Fi Protected Setup (WPS)]]. Para usar o WPS, o access point e a estação wireless (dispositivo cliente) devem ser compatíveis com WPS. Em geral, esses dispositivos terão um botão. Pressioná-los simultaneamente no access point e no adaptador associará os dispositivos usando um PIN e, em seguida, associará o adaptador ao ponto de acesso usando WPA2. O sistema gera um SSID e PSK aleatórios. Se os dispositivos não suportarem o método do botão, o PIN (impresso no WAP) poderá ser inserido manualmente.

Infelizmente, o WPS é vulnerável a um ataque de força bruta. Embora o PIN tenha oito caracteres, um dígito é para soma de verificação e o restante é verificado como dois PINs separados de quatro e três caracteres. Esses PINs separados são muitas ordens de magnitude menos resistentes à força bruta, normalmente exigindo apenas algumas horas para desvendar. Em alguns modelos, desativar o WPS por meio da interface de administração não desativa o protocolo ou não há opção para desativá-lo. Alguns APs podem bloquear um intruso se um ataque de força bruta for detectado, mas, em alguns casos, o ataque pode ser retomado quando o período de bloqueio expirar.

Para combater isso, o período de bloqueio pode ser aumentado. No entanto, isso pode deixar os APs vulneráveis a um ataque de negação de serviço (DoS). Ao provisionar um WAP, é essencial verificar quais etapas o fabricante tomou para tornar sua implementação de WPS segura e usar o nível de firmware do dispositivo necessário identificado como seguro.

O método Easy Connect, anunciado juntamente com o WPA3, destina-se a substituir o WPS como um método de configuração segura de dispositivos clientes com as informações necessárias para acessar uma rede Wi-Fi. Easy Connect é o nome comercial do protocolo de provisionamento de dispositivos (Device Provisioning Protocol, DPP).

Cada dispositivo participante deve ser configurado com um par de chaves pública/privada. O Easy Connect usa códigos de resposta rápida (Quick Response, QR) ou tags de comunicação a curta distância (Near-Field Communication, NFC) para comunicar a chave pública de cada dispositivo. Um smartphone é registrado como um aplicativo configurador Easy Connect e associado ao WAP usando seu código QR. Cada dispositivo cliente pode, então, ser associado por meio da leitura de seu QR code ou tag NFC no aplicativo de configuração. Além de solucionar os problemas de segurança relacionados ao WPS, esse método oferece uma forma simples de configurar dispositivos de Internet das Coisas (IoT) sem interface gráfica (“headless”) com conectividade Wi-Fi.

## Acesso Wi-Fi protegido 3 (WPA3)

Nem o WEP nem a versão original do WPA são considerados seguros o suficiente para uso contínuo. O WPA2 utiliza o Advanced Encryption Standard (AES), implementado no modo Counter Mode com o protocolo Cipher Block Chaining Message Authentication Code (CCM). O AES substitui o RC4 e o CCM substitui o TKIP. O CCMP fornece criptografia autenticada, que é projetada para dificultar os ataques de repetição.

As fraquezas encontradas no WPA2 levaram à substituição pretendida pelo WPA3. Os principais recursos do WPA3 são:

- [[Simultaneous Authentication of Equals (SAE)]]: substitui o protocolo de troca de chave pré-compartilhada (PSK) do WPA2, garantindo que um invasor não possa interceptar a senha do Wi-Fi mesmo capturando dados de um login bem-sucedido.
- **Enhanced Open**: criptografa o tráfego entre os dispositivos e o ponto de acesso, mesmo sem o uso de senha, aumentando a privacidade e a segurança em redes abertas.
- **Protocolos criptográficos atualizados**: substituem o [[AES CCM pelo modo de operação AES Galois Counter Mode (GCM)]]. O Galois Counter Mode (GCM) apresenta níveis de desempenho superiores ao CCM.
- **Conexão fácil de Wi-Fi**, ou Wi-Fi Easy Connect, que permite conectar dispositivos digitalizando um código QR, reduzindo a necessidade de configurações complicadas e mantendo conexões seguras.

Nota:

O desempenho do Wi-Fi também depende do suporte do dispositivo aos padrões 802.11 mais recentes. A geração mais recente (802.11ax) está sendo comercializada como Wi-Fi 6. Os padrões anteriores foram nomeados retroativamente como Wi-Fi 5 (802.11ac) e Wi-Fi 4 (802.11n). Os padrões de desempenho são desenvolvidos em paralelo com as especificações de segurança do WPA. A maioria dos dispositivos Wi-Fi 6 e alguns produtos Wi-Fi 5 e Wi-Fi 4 devem suportar WPA3 nativamente ou com uma atualização de firmware/driver.
