# Security+ CertMaster Study (Versão em português) 7.0

Para proteger uma rede, você deve confirmar que apenas usuários válidos estejam se conectando a ela. A autenticação de Wi-Fi está disponível em três tipos: pessoal, aberta e empresarial. Dentro da categoria pessoal, existem dois métodos: autenticação de chave pré-compartilhada (PSK) e autenticação simultânea de iguais (SAE).

## Autenticação de chave pré-compartilhada WPA2

No WPA2, a autenticação de [[pre-shared key (PSK)]] (Chave pré-compartilhada) usa uma senha para gerar a chave que é usada para criptografar as comunicações. Também é chamada de autenticação de grupo porque um grupo de usuários compartilha o mesmo segredo. Quando o access point é definido para o modo WPA2-PSK, o administrador configura uma senha entre 8 e 63 caracteres ASCII. Ela é convertida em um HMAC de 256 bits (expresso como um valor hexadecimal de 64 caracteres) usando o algoritmo de alongamento de chave PBKDF2. Esse HMAC é chamado de chave mestra em pares (Pairwise Master Key, PMK). O mesmo segredo deve ser configurado no ponto de acesso e em cada nó que ingressar na rede. A PMK é usada como parte da identificação mútua (handshake) de 4 vias do WPA2 para derivar várias chaves de sessão.

Nota:

Todos os tipos de autenticação pessoal de Wi-Fi mostraram ser vulneráveis a ataques que permitem ataques de dicionário ou força bruta contra a senha. A senha deve ter pelo menos 14 caracteres para mitigar os riscos de decifragem.

## Autenticação pessoal WPA3

Embora o WPA3 ainda utilize uma frase secreta para autenticar estações no modo pessoal, ele altera o método que o segredo utiliza para negociar as chaves de sessão. O esquema usado é chamado de troca de chaves autenticada por senha (Password-Authenticated Key Exchange, PAKE). No WPA3, o protocolo Simultaneous Authentication of Equals (SAE) substitui o método de chave pré-compartilhada (PSK) do WPA2 para a criação da Pairwise Master Key (PMK). No entanto, o handshake de 4 vias ainda é utilizado para a troca de chaves após o estabelecimento da PMK. A SAE usa o handshake Dragonfly, que é basicamente um acordo de chaves Diffie-Hellman sobre curvas elípticas, combinado com um valor de hash derivado da senha e do endereço MAC do dispositivo para autenticar os nós. Com a SAE, não deve haver nenhuma maneira de um invasor detectar o handshake para obter o valor do hash e tentar usar uma força bruta offline ou um ataque de dicionário para recuperar a senha. O Dragonfly também implementa chaves de sessão efêmeras que fornecem sigilo encaminhado (forward secrecy).

Nota:

As interfaces de configuração dos pontos de acesso podem usar nomenclaturas diferentes para esses métodos. Você poderá ver WPA2-Personal e WPA3-SAE em vez de WPA2-PSK e WPA3-Personal, por exemplo. Além disso, um access point pode ser configurado apenas para WPA3 ou com suporte para WPA2 legado (modo WPA3-Personal Transition). Pesquisadores identificaram falhas no WPA3-Personal quando configurado em Modo de Transição, o que permite ataques de downgrade para WPA2. ([wi-fi.org/security-update-april-2019](https://www.wi-fi.org/security-update-april-2019)).

## Autenticação avançada

Os modos de [[autenticação empresarial]] wireless, como WPA2/WPA3-Enterprise, incluem diversos componentes essenciais desenvolvidos para aumentar a segurança das redes sem fio corporativas. Um elemento importante é a autenticação 802.1x, que fornece uma estrutura de controle de acesso à rede baseada em portas, garantindo que apenas os dispositivos autenticados recebam acesso à rede. Normalmente, o 802.1x requer um servidor de autenticação, como o RADIUS (Remote Authentication Dial-In User Service ou serviço ao usuário de discagem de autenticação remota), que verifica as credenciais de usuários ou dispositivos que tentam se conectar à rede.

Nos esquemas de autenticação do modo corporativo, os usuários têm um conjunto exclusivo de credenciais, em vez da senha compartilhada usada no modo pessoal WPA2/WPA3. Exigir que cada usuário ou dispositivo seja autenticado usando credenciais exclusivas permite que os administradores de rede rastreiem o uso da rede em um nível granular. O protocolo também suporta vários tipos de protocolo de autenticação extensível (Extensible Authentication Protocol, EAP), como EAP-TLS, EAP-TTLS ou PEAP, que definem métodos de autenticação específicos. O EAP-TLS, por exemplo, usa certificados cliente-servidor para autenticação mútua, enquanto o EAP-TTLS e o PEAP utilizam um certificado do lado do servidor. O certificado do lado do servidor é usado para estabelecer um túnel seguro para transmitir credenciais de usuário e ajuda os dispositivos a validar a legitimidade do access point. A autenticação em modo corporativo inclui o gerenciamento dinâmico de chaves de criptografia, alterando automaticamente as chaves de criptografia usadas durante a sessão de um usuário.

## Remote Authentication Dial-In User Service (RADIUS) (serviço de autenticação da Internet da Microsoft)

O padrão de serviço ao usuário de discagem de autenticação remota (RADIUS) é publicado como um padrão da Internet. Existem vários produtos RADIUS para servidor e cliente.

O dispositivo NAS (Network Access Server) é configurado com o endereço IP do servidor RADIUS e com um segredo compartilhado. Isso permite que o cliente seja autenticado no servidor. Lembre-se de que o cliente é o dispositivo de acesso (switch, access point ou gateway VPN), não o computador ou notebook do usuário. Um fluxo de trabalho de autenticação RADIUS genérico ocorre da seguinte forma:

1. O dispositivo do usuário (o requerente) faz uma conexão com o dispositivo NAS, como um access point, switch ou servidor de acesso remoto.
    
2. O NAS solicita ao usuário suas credenciais de autenticação.
    
    O RADIUS suporta PAP, CHAP e EAP. A maioria das implementações agora usa o EAP, pois o PAP e o CHAP não são seguros. Se credenciais EAP forem exigidas, o NAS permite que o supplicant transmita dados [[EAP over LAN (EAPoL)]], mas não autoriza qualquer outro tipo de tráfego de rede.
    
3. O supplicant envia as credenciais como dados EAPoL.
    
    O cliente RADIUS usa essas informações para criar um pacote RADIUS Access-Request criptografado usando o segredo compartilhado. Ele envia o Access-Request para o servidor AAA (Authentication, Authorization and Accounting) utilizando UDP na porta 1812 (por padrão).
    
4. O servidor AAA descriptografa a Access-Request usando o segredo compartilhado.
    
    Se a Access-Request não puder ser descriptografada (por exemplo, porque o segredo compartilhado não está configurado corretamente), o servidor não responderá.
    
5. Com o EAP, haverá uma troca de pacotes para desafio de acesso (Access-Challenge) e solicitação de acesso (Access-Request) à medida que o método de autenticação é configurado e as credenciais verificadas.
    
    O NAS atua como elemento de passagem (pass-thru), pegando mensagens RADIUS do servidor e encapsulando-as como EAPoL para transmiti-las ao requerente.
    
6. No final desta troca, se o requerente for autenticado, o servidor AAA responde com um pacote de acesso aceito (Access-Accept); caso contrário, retorna um pacote de acesso rejeitado (Access-Reject).
    

Opcionalmente, o NAS pode usar o RADIUS para contabilidade (registro). A contabilidade usa a porta 1813. O servidor de contabilidade pode ser diferente do servidor de autenticação.
