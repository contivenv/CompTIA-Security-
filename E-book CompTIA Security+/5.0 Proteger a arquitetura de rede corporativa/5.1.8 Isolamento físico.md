# Security+ CertMaster Study (Versão em português) 7.0

 Alguns hosts são tão essenciais à segurança que não é seguro conectá-los a qualquer tipo de rede. Um exemplo é a autoridade de certificação root em PKI. Outro exemplo é um host usado para analisar a execução do malware. Um host que não está fisicamente conectado a nenhuma rede é chamado de [[air-gapped]] (isolado).

Também é possível configurar uma rede isolada, ou air-gapped. Isso significa que os hosts dentro da rede podem se comunicar, mas não há conexões com ou wireless com nenhuma outra rede. Bases militares, sites governamentais e instalações industriais usam redes (air-gapped) isoladas.

Isolar fisicamente um host ou grupo de hosts melhora a segurança, mas também acarreta desafios significativos de gerenciamento. A administração do dispositivo deve ser realizada em um terminal local. Todas as atualizações ou instalações devem ser realizadas usando USB ou mídia óptica. Essa mídia é um possível vetor de ataque e deve ser verificada antes de ser usada em um host isolado.
