

Com a [[VPN de segurança da camada de transporte (TLS)]] , o cliente se conecta ao servidor de acesso remoto usando certificados digitais. O certificado do servidor identifica o gateway de VPN para o cliente. Opcionalmente, é possível configurar o cliente usando seu próprio certificado. Isso permite a autenticação mútua, em que tanto o servidor quanto o cliente comprovam a identidade um para o outro. O TLS cria um túnel criptografado para o usuário enviar as credenciais de autenticação. Estas normalmente seriam processadas por um servidor RADIUS. Depois que o usuário for autenticado e a conexão for totalmente estabelecida, o gateway de VPN fará o tunelamento de todas as comunicações da rede local pelo soquete seguro.

Descrição

As configurações criptográficas são as seguintes: Autenticação TLS: ativada — autenticação e criptografia. Chave compartilhada TLS: caixa marcada, gerar automaticamente uma chave de autenticação TLS compartilhada. Autoridade certificadora do peer: structureality enterprise root. Lista de revogação de certificados do peer: nenhuma. Certificado do servidor: structureality remote access VPN server. Algoritmo de criptografia (obsoleto): nenhum. Algoritmo de hash de autenticação: SHA 256 (256 bits). Profundidade do certificado: um (cliente + servidor). Correspondência estrita de usuário ou CN: caixa em branco.

Nota:

Uma VPN TLS pode usar TCP ou UDP. O UDP pode ser escolhido para obter um desempenho ligeiramente superior, principalmente ao fazer o tunelamento de tráfego que requer baixa latência, como de voz ou vídeo. Já o TCP pode ser mais fácil de usar com uma política de firewall padrão. O TLS sobre UDP também é denominado Datagrama TLS (DTLS).

Nota:

É importante usar uma versão segura do TLS. No momento da elaboração deste texto, a versão mais recente é o TLS 1.3. O TLS 1.2 ainda é suportado. Versões anteriores a essa já são obsoletas.
