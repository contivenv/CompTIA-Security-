# Security+ CertMaster Study (Versão em português) 7.0

Um firewall que executa filtragem de camada de aplicativo provavelmente é implementado como um proxy. Nas situações em que um firewall de rede apenas aceita ou bloqueia tráfego, um [[servidor proxy]] funciona em um modelo store-and-forward (armazenar e encaminhar). O proxy desconstrói cada pacote, realiza análises e, se estiver em conformidade com as regras, ele reconstrói o pacote e o encaminha.

Nota:

A quantidade de reconstrução depende do proxy. Alguns proxies apenas manipulam os cabeçalhos IP e TCP. Os proxies com reconhecimento de aplicativos podem adicionar ou remover cabeçalhos HTTP. Um proxy de inspeção profunda de pacotes pode conseguir remover o conteúdo de um payload HTTP.

## Servidor proxy de encaminhamento

Um proxy de encaminhamento atua no tráfego de saída de um protocolo específico. Por exemplo, um proxy Web permite que os computadores clientes na rede local se conectem a sites e endereços seguros na Internet. Este é um proxy de encaminhamento que atende as portas TCP 80 e 443 para tráfego de saída.

O principal benefício de um proxy é que os computadores clientes se conectam a um ponto especificado da rede de perímetro para acesso à Web. Isso proporciona um determinado grau de gerenciamento e segurança do tráfego. Além disso, a maioria dos servidores proxy web oferece [[mecanismos de cache]], nos quais páginas web frequentemente solicitadas são armazenadas no proxy, eliminando a necessidade de buscar essas páginas novamente em requisições subsequentes.

Um servidor proxy deve compreender a que aplicativo está atendendo. Por exemplo, um proxy Web deve ser capaz de analisar e modificar comandos HTTP e, também possivelmente, scripts e código da página da Web. Alguns servidores proxy são específicos para um aplicativo, enquanto outros são multiuso. Um proxy multiuso é configurado com filtros para vários tipos de protocolo, como dados da Web, dados do protocolo de transferência de arquivos e dados do protocolo de e-mail.

Os servidores proxy podem ser classificados em termos gerais como não transparentes ou transparentes.

- [[No caso do proxy não transparente]], o cliente precisa ser configurado com o endereço do servidor proxy e o número da porta para usá-lo. Por convenção, a porta na qual o servidor proxy aceita conexões de cliente é geralmente configurada como porta TCP/8080.
- Um [[proxy transparente (ou forçado ou interceptador)]] intercepta o tráfego do cliente sem que o cliente tenha de ser reconfigurado. Um proxy transparente deve ser implementado como um roteador ou um dispositivo de rede em linha.

Nota:

Um script de configuração automática de proxy (proxy auto-configuration - PAC) permite que um cliente defina as configurações de proxy sem a intervenção do usuário. O protocolo Web Proxy Auto-discovery (WPAD) (descoberta automática de proxy da Web) permite que os navegadores localizem um arquivo PAC.

## Servidores proxy reversos

Um servidor [[proxy reverso]] fornece tráfego de entrada específico do protocolo. Um proxy reverso geralmente é implantado na borda da rede e configurado para atender solicitações de clientes de uma rede pública (Internet). O proxy aplica regras de filtragem e, se autorizado, cria a solicitação apropriada e a encaminha a um servidor de aplicativos em uma zona de sub-rede especialmente protegida na rede local.
