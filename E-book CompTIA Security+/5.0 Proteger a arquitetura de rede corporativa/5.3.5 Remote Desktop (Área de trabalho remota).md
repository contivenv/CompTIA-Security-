# Security+ CertMaster Study (Versão em português) 7.0

Uma VPN de acesso remoto une o computador ou smartphone do usuário a uma rede privada remota por meio de um túnel seguro em uma rede pública. O acesso remoto também pode ser um meio para se conectar a um computador específico em uma rede. Esse tipo de acesso remoto envolve conectar-se a um servidor de terminal em um host usando um software que transfere apenas dados em shell. A conexão pode ser um cliente e um servidor de terminal na mesma rede local ou em redes remotas.

Uma ferramenta gráfica de acesso remoto envia dados da tela e do áudio do host remoto para o cliente e transfere os comandos do mouse e do teclado do cliente para o host remoto. O [[Remote Desktop Protocol (RDP) (Protocolo de área de trabalho remota)]] da Microsoft pode ser usado para acessar uma máquina física de forma individual.

Como alternativa, um site pode operar um gateway de área de trabalho remota que facilita o acesso a áreas de trabalho virtuais ou aplicativos individuais em execução nos servidores de rede. As conexões RDP são criptografadas por padrão. Existem várias alternativas populares para áreas de trabalho remotas. A maioria permite acesso remoto a plataformas que não são Windows (macOS e iOS, Linux, Chrome OS e Android, por exemplo). Alguns exemplos são o TeamViewer ([teamviewer.com/pt-br](https://www.teamviewer.com/en/)) e [[Virtual Network Computing (VNC) (Computação em rede virtual)]], que é implementado por vários provedores diferentes (de destacar, o [realvnc.com/pt](https://www.realvnc.com/en/)).

No passado, esses produtos de área de trabalho remota exigiam um aplicativo cliente dedicado. Hoje, o acesso remoto à área de trabalho pode ser feito apenas utilizando um navegador web. O elemento canvas introduzido em HTML5 permite que um navegador desenhe e atualize uma área de trabalho com atraso relativamente reduzido. Ele também pode lidar com áudio. Isso é chamado de [[VPN HTML5]] ou gateway de área de trabalho remota sem cliente ([guacamole.apache.org](https://guacamole.apache.org/)). Essa solução usa um protocolo chamado WebSocket, que permite o envio de mensagens bidirecionais entre servidor e cliente sem exigir a sobrecarga de solicitações HTTP separadas.
