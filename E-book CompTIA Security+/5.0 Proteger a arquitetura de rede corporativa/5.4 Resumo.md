# 

Você deve entender o uso da segmentação para implementar topologias de segurança baseadas em zonas e saber selecionar os tipos de controle e locais de implantação apropriados para proteger redes e hosts em determinados cenários.

**Diretrizes para implementar a arquitetura de rede corporativa segura**

Siga estas diretrizes ao implementar designs de redes novas ou estendidas:

- Identifique os fluxos de trabalho empresariais e os servidores, clientes, protocolos e ativos de dados que os suportam. Crie blocos ou zonas de rede segmentados que atendam aos requisitos de segurança, utilizando VLANs, sub-redes e políticas de firewall para implementar o conceito.
- Implante dispositivos e protocolos de comutação e roteamento para suportar cada bloco, considerando a segurança da porta ou o controle de acesso à rede (NAC) 802.1x.
- Analise a superfície de ataque e selecione controles eficazes implantados em locais de rede apropriados:
    - Implante segurança da porta ou NAC 802.1x para reduzir os riscos de dispositivos não autorizados conectados a portas de rede físicas.
        
    - Implante firewalls de roteamento para reforçar o controle de acesso e a prevenção contra invasões nos perímetros da zona, utilizando filtragem de camada 4 ou camada 7 conforme apropriado.
        
    - Implante firewalls transparentes para proteger hosts e segmentos sem precisar alterar a topologia de IP.
        
    - Implante sensores para detecção de invasões pós-firewalls ou em uma porta de switch espelhada para monitorar o tráfego em uma zona.
        
    - Considere o uso de servidores proxy, NGFW e WAF para filtragem avançada conhecedora de usuários e aplicativos.
        
    - Considere o uso de UTM para implantar recursos adicionais de controle de segurança juntamente com a funcionalidade de firewall/IPS.
        
- Desenvolva serviços com carga balanceada para fornecer alta disponibilidade e tolerância a falhas.
- Atenda aos requisitos de comunicações seguras para acesso remoto cliente a site, site a site e host a host, utilizando os protocolos de VPN TLS e IPsec.
- Implemente comunicações seguras para acesso remoto de área de trabalho/shell com RDP e SSH. Considere o uso de um jump server para consolidar o gerenciamento de um grupo de servidores em uma zona protegida.
