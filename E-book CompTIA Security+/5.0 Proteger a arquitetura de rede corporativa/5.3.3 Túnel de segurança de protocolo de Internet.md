# Security+ CertMaster Study (Versão em português) 7.0

O protocolo Transport Layer Security (TLS) (segurança da camada de transporte) é aplicado ao aplicativo. [[O Internet Protocol Security (IPsec) (protocolo de segurança IP)]] opera na camada de rede do modelo OSI (camada 3). Isso significa que ele pode ser implementado sem precisar configurar o suporte a aplicativos específicos. Além disso, ele incorre em menos sobrecarga de pacotes.

Existem dois protocolos principais no IPsec que podem ser aplicados individualmente ou em conjunto, dependendo da política:

- [[Protocolo Authentication Header (AH) (cabeçalho de autenticação)]]: executa um hash criptográfico em todo o pacote, incluindo o cabeçalho IP, além de uma chave secreta compartilhada (conhecida apenas pelos hosts comunicantes), e adiciona esse valor ao cabeçalho como um Integrity Check Value (ICV) (valor de verificação de integridade). O destinatário executa a mesma função no pacote e na chave e deve obter o mesmo valor para confirmar que o pacote não foi modificado. O payload não é criptografado, portanto, esse protocolo não fornece confidencialidade.
- [[Encapsulating Security Payload (ESP) (encapsulamento de segurança de payload)]]: pode ser usado para criptografar o pacote em vez de simplesmente calcular um ICV. O ESP anexa três campos ao pacote: um cabeçalho, um trailer (fornecendo preenchimento para a função criptográfica) e um valor de verificação de integridade. Ao contrário do AH, o ESP exclui o cabeçalho IP ao calcular o ICV.

O IPsec pode ser usado em dois modos:

- **Modo de transporte**: é usado para proteger as comunicações entre hosts em uma rede privada. Quando o ESP é aplicado no modo de transporte, o cabeçalho IP de cada pacote não é criptografado, apenas os dados de payload. Se o AH for usado no modo de transporte, ele poderá oferecer integridade ao cabeçalho IP.

Figura 1. Datagrama de IPsec usando AH e ESP no modo de transporte.  
![Datagrama IPsec com os 4 segmentos: cabeçalho IP; AH, ICV; ESP (TCP/UDP e payload); e Trailer.](https://cdn.testout.com/pt-br-security-plus-v7-study-pt-br/materials/resources/text/s_secure_communications/pt-br_9434-1692974865939.png)  

- **Modo de túnel** : é usado para comunicações entre sites VPN através de uma rede não segura. Com o ESP, todo o pacote IP (cabeçalho e payload) é criptografado e encapsulado como um datagrama com um novo cabeçalho IP. Não há caso de uso para o AH no modo de túnel, pois geralmente é necessária confidencialidade.
