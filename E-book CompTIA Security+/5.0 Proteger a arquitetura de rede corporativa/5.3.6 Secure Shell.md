# Security+ CertMaster Study (Versão em português) 7.0

O [[Secure Shell (SSH)]] é o principal meio de obter acesso remoto seguro a um terminal de linha de comando. O SSH é usado principalmente para administração remota e transferência segura de arquivos (SFTP). Inúmeros produtos SSH comerciais e de código aberto estão disponíveis para as principais plataformas NOS. O mais utilizado é o OpenSSH ([openssh.com](https://www.openssh.com/)).

Os servidores SSH são identificados por um par de chaves pública/privada denominado chave host. Os nomes de host podem ser manualmente mapeados às chaves de host por cada cliente SSH ou através dos diversos produtos de software empresariais desenvolvidos para o gerenciamento de chaves de host SSH.

Nota:

A chave do host deverá ser alterada se houver qualquer suspeita de comprometimento do host. Se um invasor obtiver a chave privada de um servidor ou dispositivo, ele poderá se disfarçar desse servidor ou dispositivo e realizar um ataque de spoofing (falsificação), geralmente com o objetivo de obter outras credenciais de rede.

A chave do host do servidor é usada para configurar um canal seguro para que o cliente possa enviar credenciais de autenticação.

## Autenticação do cliente SSH

O SSH permite que o cliente use vários métodos para autenticar no servidor. Cada um desses métodos pode ser ativado ou desativado conforme a necessidade no servidor usando o arquivo/etc/ssh/sshd_config:

- **Nome de usuário/senha**: é quando o cliente envia credenciais que são verificadas pelo servidor SSH em uma base de dados local do usuário ou usando um servidor RADIUS.
- **Autenticação por chave pública**: é quando a chave pública de cada usuário remoto é adicionada a uma lista de chaves autorizadas para cada conta local no servidor SSH.
- **Kerberos**: é quando o cliente envia as credenciais Kerberos (um TGT, tíquete de concessão de tíquete) obtidas quando o usuário faz login na estação de trabalho no servidor usando a GSSAPI (Generic Security Services Application Program Interface). O servidor SSH entra em contato com o Serviço de concessão de tíquete (em um ambiente Windows, será um controlador de domínio) para validar as credenciais.

Nota:

O gerenciamento de chaves públicas de cliente válidas é uma tarefa crítica de segurança. Muitos ataques recentes a servidores web exploraram o gerenciamento inadequado de chaves. Se a chave privada de um usuário estiver comprometida, exclua a chave pública do dispositivo e, em seguida, gere outro par de chaves no dispositivo cliente (corrigido) do usuário e copie a chave pública para o servidor SSH. Sempre exclua as chaves públicas quando as permissões de acesso do usuário forem revogadas.

## Comandos SSH

Os comandos SSH são usados para se conectar a hosts e configurar métodos de autenticação. Para se conectar a um servidor SSH em 10.1.0.10 usando uma conta chamada “bobby” e autenticação por senha, execute:

`ssh bobby@10.1.0.10`

Os seguintes comandos criam um novo par de chaves e o copiam para uma conta no servidor remoto:

`ssh-keygen -t rsa`

`ssh-copy-id bobby@10.1.0.10`

Em um prompt SSH, agora é possível usar os comandos de shell padrão do Linux. Use exit para encerrar a conexão.

Você pode usar o comando scp para copiar um arquivo do servidor remoto para o host local:

`scp bobby@10.1.0.10:/logs/audit.log audit.log`

Reverta os argumentos para copiar um arquivo do host local para o servidor remoto. Para copiar o conteúdo de um diretório e subdiretórios (recursivamente), use a opção`-r`.
