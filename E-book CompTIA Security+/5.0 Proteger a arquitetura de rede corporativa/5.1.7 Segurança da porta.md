

Cada porta de parede e de switch é uma oportunidade para um invasor conectar um dispositivo à rede. Um agente de ameaças que pode operar um host com acesso físico a um segmento de rede pode lançar diversos ataques.

O acesso às portas físicas de switch e ao hardware de switch deve ser restrito ao pessoal autorizado. Para isso, coloque os dispositivos de switch em salas de servidor seguras e/ou gabinetes de hardware com chave. Para evitar a conexão de dispositivos clientes não autorizados em portas de parede não seguras, a porta de switch à qual o cabeamento da porta de parede se conecta pode ser desativada administrativamente ou o cabo de conexão pode ser fisicamente removido da porta de switch. Desativar completamente as portas dessa maneira pode gerar uma sobrecarga administrativa muito grande e dar margem a erros. Além disso, não fornece proteção completa, pois um invasor pode desconectar um dispositivo de uma porta ativada e conectar seu próprio equipamento. Consequentemente, métodos mais sofisticados para garantir a [[segurança de portas]] foram desenvolvidos.

## Filtragem MAC e limitação MAC

O adaptador de rede em cada computador host é identificado por um endereço MAC. Configurar a [[filtragem MAC]] significa que uma porta de switch só permite a conexão de determinados endereços MAC. Isso pode ser feito criado uma lista de endereços MAC válidos ou especificando um limite para o número de endereços permitidos. Por exemplo, se a segurança da porta estiver ativada para um máximo de dois endereços MAC, o switch registrará os dois primeiros a se conectarem a essa porta. O switch então descartará qualquer tráfego de máquinas com endereços MAC diferentes que tentarem se conectar.

## 802.1x e Protocolo de Autenticação Extensível

A restrição do acesso por endereço MAC é difícil de gerenciar e ainda é propenso a spoofing. É possível aumentar a segurança exigindo que os computadores e/ou usuários se autentiquem antes de terem acesso total à rede. O padrão [[IEEE 802.1X]] Port-based Network Access Control (PNAC) permite que um switch exija autenticação quando um host se conecta a uma de suas portas. O 802.1X utiliza arquitetura de autenticação, autorização e contabilização (AAA):

- [[Solicitante]]: é o dispositivo que solicita acesso, como o PC ou laptop de um usuário.
- [[Autenticador]]: é o dispositivo de comutação (ou qualquer tipo de appliance de acesso à rede). Ele não valida as solicitações de autenticação diretamente, mas atua como um condutor para os dados de autenticação.
- **Servidorde autenticação**: é o servidor que mantém ou pode acessar um diretório de objetos de rede e que pode validar solicitações de autenticação, emitir autorizações e realizar a contabilização de eventos de segurança.

O padrão 802.1X é implementado por dois protocolos:

- [[Extensible Authentication Protocol (EAP) (Protocolo de autenticação extensível)]]: fornece uma estrutura para implantar vários tipos de métodos de autenticação. É frequentemente usado com certificados digitais para estabelecer uma relação de confiança e criar um túnel seguro para transmitir as credenciais do usuário ou para realizar a autenticação de smart-card (cartão inteligente) sem uma senha.
- [[Remote Authentication Dial-In User Service (RADIUS)]]: permite que o autenticador e o servidor de autenticação comuniquem as decisões de autenticação e autorização. O autenticador é um cliente RADIUS e o servidor de autenticação é um servidor RADIUS.

Quando um host se conecta a uma porta de switch habilitada para 802.1x, o switch abre a porta apenas para o protocolo EAP over LAN (EAPoL). A porta do switch só permite acesso total aos dados quando o host for autenticado. O switch recebe um pacote EAP com as credenciais do suplicante. Estas são criptografadas e não podem ser lidas pelo switch. O switch usa o protocolo RADIUS para enviar o pacote EAP ao servidor de autenticação. Esse servidor pode acessar o diretório de contas de usuários e validar as credenciais. Se a autenticação for bem-sucedida, ele informará ao switch que o acesso total à rede pode ser concedido.

Descrição

Os primeiros passos são:

1. O cliente e o servidor RADIUS são pré-configurados com o mesmo segredo compartilhado

2. O suplicante se conecta à rede

3. O switch habilita o EAPoL e instrui o requerente a se autenticar

4. O suplicante transmite a dados EAP

5. O switch criptografa os dados EAP usando o segredo compartilhado e o encaminha para o servidor RADIUS

6. O servidor e o cliente RADIUS são pré-configurados com o mesmo segredo compartilhado.

7. O servidor RADIUS gera um Access-Accept

8. O switch descriptografa o Access-Accept e abre o canal da rede para o tráfego regular
