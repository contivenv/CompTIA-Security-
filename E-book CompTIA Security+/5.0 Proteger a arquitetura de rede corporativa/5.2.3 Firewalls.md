

Firewall é um controle preventivo projetado para aplicar políticas ao tráfego que entra e sai de uma zona de rede.

## Filtragem de pacotes

Um [[firewall de filtragem de pacotes]] é configurado especificando um grupo de regras chamado Access Control List (ACL) (Lista de controle de acesso). Cada regra define um tipo específico de pacote de dados e a ação a ser realizada quando um pacote corresponder à regra. Um firewall de filtragem de pacotes pode inspecionar os cabeçalhos dos pacotes IP.As regras são baseadas nas informações encontradas nesses cabeçalhos:

- **Filtragem de IP**: aceita ou nega o tráfego com base nos bits do endereço IP de origem e/ou destino. A maioria dos firewalls pode filtrar por endereços MAC.
- **Protocol ID/type**: é um pacote IP que carrega um protocolo identificado. É mais comum que sejam dados TCP ou UDP. Outros tipos incluem tráfego de diagnóstico de Internet Control Message Protocol (ICMP) (Protocolo de mensagem de controle de Internet) e protocolos que facilitam o roteamento.
- **Filtragem de portas/segurança**: aceita ou nega um pacote com base nos números da porta TCP/UDP de origem e destino.

Se a ação estiver configurada para aceitar ou permitir, o firewall permitirá a passagem do pacote. Uma ação de descarte ou negação descarta silenciosamente o pacote. Uma ação de rejeição bloqueia o pacote, mas responde ao remetente com uma mensagem ICMP, como “porta inacessível”.

ACLs separadas filtram o tráfego de entrada e de saída.O controle do tráfego de saída pode impedir que aplicativos não autorizados sejam executados na rede e combater malware, como backdoors.

## Posicionamento e atributos do dispositivo de firewall

Os firewalls podem ser implementados como diversos tipos de dispositivos de hardware ou como software executado em um host de computação geral. Alguns tipos de firewall são mais adequados para bordas de rede ou zonais; outros são projetados para proteger hosts individuais.

Um [[firewall de dispositivo]] é um firewall de hardware autônomo implantado para monitorar o tráfego que entra e sai de uma zona de rede. Ele pode ser implantado de três formas:

- **Roteador (camada 3)**: o firewall realiza o encaminhamento entre sub-redes. Cada interface no firewall se conecta a uma sub-rede diferente e representa uma zona de segurança diferente. Cada interface é configurada com um endereço IP e MAC.
- **Em modo bridge (camada 2)**: o firewall inspeciona o tráfego que passa entre dois nós, como um roteador e um switch. Ele conecta as interfaces Ethernet entre os dois nós, funcionando como um switch. Apesar de realizar o encaminhamento na camada 2, o firewall ainda pode inspecionar e filtrar o tráfego com base na gama completa de cabeçalhos de pacote. As interfaces do firewall são configuradas com endereços MAC, mas não com endereços IP.
- **Inline ou em linha (camada 1)**: o firewall atua como um segmento de cabo. As duas interfaces não têm endereços MAC ou IP. O tráfego recebido em uma interface é bloqueado ou encaminhado pela outra interface. Esse método também é chamado de virtual wire ou bump-in-the-wire.

Os modos de firewall bridge e inline/em linha podem ser referidos como modos transparentes. O caso de uso típico de um modo transparente é implantar um firewall sem precisar reconfigurar sub-redes e reatribuir endereços IP em outros dispositivos. Por exemplo, você pode implantar um firewall transparente na frente de um host de servidor Web sem precisar alterar o endereço IP do host. Ou então, ele pode ser implantado entre um roteador e um switch.

Nota:

Um firewall transparente precisa de uma interface adicional para gerenciamento e configuração. Essa interface tem um endereço IP. Um firewall roteado pode ter uma interface dedicada de gerenciamento ou aceitar tráfego de gerenciamento em qualquer interface. Usar uma interface dedicada de gerenciamento é mais seguro.

Um appliance [[roteador firewall]] ou firewall router implementa a funcionalidade de filtragem como parte do firmware do roteador. A diferença é que um dispositivo roteador serve principalmente para roteamento, tendo o firewall como funcionalidade secundária. Roteadores/modems de Internet para Pequenos Escritórios ou Escritórios em Casa (SOHO) já vêm com firewall embutido, por exemplo.
