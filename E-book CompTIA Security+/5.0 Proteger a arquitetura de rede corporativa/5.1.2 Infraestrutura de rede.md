

É útil empregar um modelo em camadas para analisar a infraestrutura e os serviços de rede. O modelo Open Systems Interconnection (Interconexão de sistemas abertos - OSI) é um exemplo amplamente citado de como definir camadas de funções de rede.

A rede é composta de nós e links. Na camada física (PHY), ou camada 1 no modelo OSI, os links são implementados como cabos de par trançado que transmitem sinais elétricos, cabos de fibra óptica que transportam sinais de luz infravermelha ou como dispositivos wireless que transmitem ondas de rádio.

Há dois tipos de nós. Um nó host é aquele que inicia as transferências de dados. Os hosts geralmente são servidores ou clientes. Um nó intermediário encaminha o tráfego pela rede. Esse encaminhamento ocorre em camadas diferentes e com escopos diferentes. Um escopo de rede de um único local é chamado de rede de área local (LAN). Redes que abrangem âmbitos metropolitanos, nacionais ou globais são chamadas de redes de longa distância (WANs).

Cada nó de rede deve ser identificável por meio de um endereço exclusivo. Essa função de endereçamento também ocorre em camadas diferentes com escopos diferentes.

As funções de encaminhamento e endereçamento são processadas pelos seguintes dispositivos e protocolos de rede:

- Os switches encaminham quadros entre os nós de uma rede cabeada. O adaptador de rede em cada host é conectado a uma porta de switch por meio de um cabo. Os switches funcionam na camada 2 do modelo OSI. A maioria das LANs usa redes baseadas no padrão Ethernet. Um switch Ethernet toma decisões de encaminhamento com base no endereço do hardware ou do controle de acesso ao meio (media access control - MAC) dos hosts conectados. Um endereço MAC é um valor de 48 bits escrito em notação hexadecimal, como 00-15-5D-01-CA-4A. Esse endereçamento funciona apenas no segmento de rede local. Isso é chamado de domínio de transmissão.

Descrição

O modelo inclui sete camadas: Camada 1: Física, Camada 2: Enlace de Dados, Camada 3: Rede, Camada 4: Transporte e Camada 7: Aplicação. A camada de enlace de dados inclui um switch e um ponto de acesso. Ela possui o endereço MAC 00-15-5D-01-CA-4A e o VLAN ID 100. A camada de rede inclui um roteador e um protocolo de internet. Ela possui o endereço IP 10.1.0.192 e a sub-rede 10.1.0.0/24. O Protocolo de Resolução de Endereços (ARP) ocorre entre a camada de enlace de dados e a camada de rede. A camada de transporte inclui o Transmission Control Protocol (TCP) e o User Datagram Protocol (UDP). A camada de aplicação inclui um servidor e protocolos de aplicação como HTTP, SMTP, SMB, FTP e DNS. Ela possui um nome de domínio totalmente qualificado (FQDN), www.515support.com. O sistema de nomes de domínio (DNS) sobrepõe as camadas de transporte e aplicação.

- Os access points em redes wireless fornecem uma ponte entre uma rede com fio e hosts ou estações wireless. Os access points funcionam na camada 2 do modelo OSI. Os dispositivos wireless também usam o endereçamento MAC na camada 2.
- Os roteadores enviam pacotes por uma rede da Internet, tomando decisões de encaminhamento com base em endereços de Protocolo de Internet (IP). Os roteadores funcionam na camada 3 do modelo OSI. Cada segmento local normalmente terá um roteador conectado a ele. O roteador atua como um gateway padrão para os hosts no segmento, sendo utilizado para enviar pacotes a outros segmentos.
- Os protocolos de transporte permitem que os clientes troquem dados com os servidores de aplicativos. O protocolo de controle de transmissão (Transmission Control Protoco - TCP) estabelece conexões confiáveis, enquanto o protocolo de datagrama de usuário (User Datagram Protocol - UDP) permite transferências não confiáveis e sem conexão. Cada protocolo de aplicativo é identificado por uma porta TCP ou UDP. Essa funcionalidade é definida na camada 4 do modelo OSI.
- Os protocolos de aplicativo possibilitam a funcionalidade cliente/servidor para serviços ao nível do usuário, como navegação na Web, email e transferência de arquivos. Os protocolos de aplicativos funcionam na camada 7 do modelo OSI.
- Os servidores do sistema de nomes de domínio (Domain Name System - DNS) hospedam registros de nomes e executam a resolução de nomes para permitir que aplicativos e usuários abordem hosts e serviços usando nomes de domínio completamente qualificados (Fully Qualified Domain Name — FQDNs) em vez de endereços IP. O DNS também funciona na camada 7 do modelo OSI, mas é um serviço de infraestrutura, e não um serviço ao nível do usuário, como é a navegação na Web.

Nota:

O modelo OSI tem três camadas superiores. Em termos práticos, não é tão útil distinguir as funções das camadas 5, 6 e 7, então pense apenas em aplicativos que funcionam na camada 7.
