

O encaminhamento de camada 3, ou roteamento, aplica um esquema de endereçamento lógico para identificar cada rede. A arquitetura de camada 3 representa a [[segmentação lógica]] de redes e a criação de redes dentro de redes, ou sub-redes (subnets). Cada sub-rede é um domínio de transmissão separado. Na camada 3, os nós são identificados por endereços de Protocolo de Internet (IP) e os links são identificados por rotas.

## Protocolo de Internet

O [[Protocolo de Internet (IP)]] fornece o mecanismo de endereçamento para redes e sub-redes lógicas. Um endereço IP versão 4 (IPv4) de 32 bits é escrito em notação decimal pontuada, com um prefixo de rede ou máscara de sub-rede para dividir o endereço em partes de ID de rede e ID de host. Por exemplo, no endereço IP 10.1.1.0/24, o prefixo /24 indica que os primeiros três quartos do endereço (10.1.1.x) são o ID da rede, enquanto o restante identifica exclusivamente um host nessa rede. Esse prefixo /24 também pode ser escrito como uma máscara de sub-rede na forma 255.255.255.0.

Na arquitetura de rede hierárquica, cada bloco de acesso pode ser designado como uma sub-rede IP separada. Esse sistema de endereçamento lógico de camada 3 facilita a criação de regras de controle de acesso para determinar o tráfego que pode fluir entre blocos ou zonas.

Descrição

Um roteador ou switch de camada 3 está conectado a um roteador de borda ou firewall por meio de um proxy firewall inline que possui a sub-rede 10.1.128.0/24. Ele também está conectado a três switches de acesso pelas sub-redes 10.1.48.0/24, 10.1.32.0/24 e 10.1.16.0/24. O roteador de borda ou firewall está conectado a um switch de acesso através da sub-rede 192.168.42.0/24. O switch de acesso está conectado a um ponto de acesso Wi-Fi, que, por sua vez, está conectado à rede de convidados.

As redes também podem usar o endereçamento IPv6 de 128 bits. Os endereços IPv6 são escritos usando notação hexadecimal no formato geral: 2001:db8::abc:0:def0:1234. No IPv6, os últimos 64 bits são fixados como o ID da interface do host. Os primeiros 64 bits contêm informações de rede em uma hierarquia definida. Por exemplo, os roteadores de um ISP podem usar os primeiros 48 bits para determinar onde a rede está hospedada na Internet global. Dentro dessa rede, o administrador do site pode usar os 16 bits restantes (dos 64) para dividir a rede local em sub-redes.

Nota:

Um pacote enviado via IP deve ser encaminhado usando endereçamento de camada 2. O IPv4 usa o Address Resolution Protocol (ARP) (Protocolo de resolução de endereços) para mapear a interface IP de um host a um endereço MAC. O IPv6 usa o protocolo Neighbor Discovery (ND - Descoberta de vizinhos) para a mesma finalidade.

## LANs virtuais

O mapeamento da topologia lógica de IP para switches de hardware físicos nem sempre é um processo simples. Esse problema é resolvido pelo recurso de [[LAN virtual (VLAN)]] suportado pela maioria dos switches. Todos os switches conectados juntos na mesma rede local podem ser configurados com um conjunto coerente de IDs de VLAN. Um ID de VLAN é um valor de 2 a 4.094. Qualquer porta de switch pode ser atribuída a uma VLAN específica, independentemente da localização física do switch. Diferentes portas no mesmo switch podem ser atribuídas a diferentes VLANs.

Cada VLAN é um domínio separado de Camada 2. O tráfego enviado entre VLANs que estão conectadas ou mapeadas para uma entidade de Camada 3, como um roteador com interfaces sub-redes, deve passar pelo dispositivo de Camada 3 para ser encaminhado. Isso significa que a segmentação imposta pelas VLANs na camada 2 pode ser mapeada a divisões lógicas definidas por sub-redes IP na camada 3.

O roteador está conectado à VLAN 16 pela sub-rede 10.1.16.0/24 e ao switch de acesso. Também está conectado à VLAN 24 pela sub-rede 10.1.24.0/24 e ao switch de acesso. O roteador de borda ou firewall está conectado a um switch de acesso que possui a sub-rede 192.168.42.0/24. O switch de acesso está conectado a um ponto de acesso Wi-Fi, que, por sua vez, está conectado à rede de convidados.

No diagrama acima, o bloco de acesso para dispositivos clientes utiliza duas VLANs para segmentar hosts de computadores de estação de trabalho (VLAN32) dos aparelhos de Voz sobre Protocolo de Internet (Voice over Internet Protocol - VoIP) (VLAN40). As VLANs são mapeadas a duas sub-redes: 10.1.32.0/24 e 10.1.40.0/24.

Um aparelho VoIP com endereço IP 10.1.40.100 precisaria usar um roteador para se comunicar com um host de computador com IP 10.1.32.100. Os dois hosts podem estar conectados ao mesmo switch, mas a configuração da VLAN os impede de se comunicar um com o outro diretamente. Além disso, uma regra de controle de acesso configurada no roteador poderia impedir esse tipo de comunicação caso fosse considerado um risco à segurança.

A topologia da VLAN pode ser estendida por vários switches. Imagine uma situação em que o escritório se expande para o segundo andar, demandando um dispositivo de switch adicional para fornecer portas suficientes. Os mesmos IDs e sub-redes de VLAN podem ser configurados para o segundo andar, tornando os dispositivos nos dois andares parte dos mesmos dois segmentos de estação de trabalho e VoIP.
