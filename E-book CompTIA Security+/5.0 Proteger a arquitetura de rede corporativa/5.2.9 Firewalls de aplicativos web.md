# Security+ CertMaster Study (Versão em português) 7.0

Um [[firewall de aplicações Web (web application firewall - WAF)]] foi desenvolvido para proteger o software executado em servidores Web e seus bancos de dados back-end contra ataques de injeção de código e de negação de serviço. Os WAFs usam regras de processamento com reconhecimento de aplicativo para filtrar o tráfego e executar a detecção de intrusão específica para o aplicativo. O WAF pode ser programado com assinaturas de ataques conhecidos e usar correspondência de padrões para bloquear solicitações que contenham código suspeito. Os resultados de um WAF serão registrados em um log, podendo revelar possíveis ameaças ao aplicativo Web.

Figura 1. Com o WAF ModSecurity instalado neste servidor IIS, uma tentativa de varredura foi detectada e registrada como um evento de aplicativo. Como você pode ver, o conjunto de regras padrão gera muitos eventos.

Descrição

A subopção “application” em logs do Windows está selecionada no menu à esquerda. O painel à direita possui duas seções. A seção superior, intitulada application, lista o número de eventos: 12.660. Ela apresenta o nível, data e hora e fonte da aplicação em formato de tabela. A seção inferior, intitulada Evento 1, Mod Security, possui duas abas: Geral e Detalhes (selecionada). Há dois botões de opção: Friendly View (selecionado) e XML View. Abaixo, há um código de computador.

Um WAF pode ser implantado como um dispositivo de proteção da zona em que o servidor Web se encontra ou como um software plug-in em uma plataforma de servidor Web.
