

Um firewall de filtragem de pacotes básico é stateless, ou seja, não tem monitoração de estado. Isso significa que ele não preserva informações sobre sessões de rede. Cada pacote é analisado de forma independente, sem nenhum registro de pacotes processados anteriormente. Este tipo de filtragem requer esforço mínimo de processamento, mas pode ser vulnerável a ataques espalhados por uma sequência de pacotes. Um firewall sem monitoração de estado também pode introduzir problemas no fluxo de tráfego, principalmente ao usar algum tipo de balanceamento de carga ou quando clientes ou servidores precisam usar portas atribuídas dinamicamente.

Um firewall de [[inspeção stateful]] (com monitoração de estado) rastreia informações sobre a sessão estabelecida entre dois hosts. Todos os firewalls agora incorporam algum nível de capacidade de inspeção de estado. Os dados das sessões são armazenados em uma [[tabela de estado]]. Quando um pacote chega, o firewall o verifica para confirmar se ele pertence a uma conexão existente. Em caso negativo, ele aplica as regras comuns de filtragem de pacotes para determinar se deve permitir. Depois que a conexão é autorizada, o firewall geralmente permite a passagem do tráfego sem monitoramento para poupar esforço de processamento.

## Firewall de Camada 4

A camada 4 é a camada de transporte OSI. Um [[firewall de camada 4]] examina o handshake de três vias do TCP para distinguir as conexões novas das já estabelecidas. Uma conexão TCP legítima deve seguir uma sequência SYN > SYN/ACK > ACK para estabelecer uma sessão, que é então rastreada usando números de sequência. O que divergir disso, como SYN sem ACK ou anomalias no número de sequência, podem ser descartado como flooding mal-intencionado ou tentativa de sequestro de sessão. O firewall pode ser configurado para responder a esses ataques bloqueando os endereços IP de origem e limitando as sessões. Ele também pode rastrear o tráfego UDP, embora isso seja mais difícil, visto que o UDP é um protocolo sem conexões. Também é provável que consiga detectar anomalias no cabeçalho IP e no ICMP.

## Firewall de Camada 7

Um [[firewall de camada 7]] pode inspecionar os cabeçalhos e o conteúdo (payload) dos pacotes da camada de aplicação. Um recurso importante é verificar se o protocolo do aplicativo corresponde à porta, pois o malware pode tentar enviar dados TCP brutos pela porta 80 apenas por ela estar aberta, por exemplo. Outro exemplo, um firewall de aplicativo Web poderia analisar os cabeçalhos HTTP e o código de formatação da página da Web presentes em pacotes HTTP para identificar strings que correspondam a algum padrão em seu banco de dados de ameaças.

Os firewalls com reconhecimento de aplicativos têm muitos nomes diferentes, incluindo gateway de camada de aplicativo, inspeção stateful multicamada e inspeção profunda de pacotes. Os dispositivos com reconhecimento de aplicativos devem ser configurados com filtros separados para cada tipo de tráfego (HTTP e HTTPS, SMTP/POP/IMAP, FTP e assim por diante).
