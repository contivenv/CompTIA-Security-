# Security+ CertMaster Study (Versão em português) 7.0

Um sistema de detecção de intrusão é um controle que realiza a análise em tempo real do tráfego de rede ou dos logs de sistema e aplicação.

## Sensores

Um sistema de detecção de intrusão em rede captura o tráfego por meio de um analisador de pacotes, chamado de sensor. O sensor pode usar um SPAN/porta espelhada ou um TAP em linha. Normalmente, o sensor de captura de pacotes é colocado atrás de um firewall ou próximo a um servidor particularmente importante. A ideia geralmente é identificar tráfego malicioso que tenha conseguido passar pelo firewall. Um único sniffer pode registrar uma grande quantidade de dados de tráfego, portanto, não se deve inserir vários sensores por toda a rede sem fornecer os recursos para gerenciá-los adequadamente. Dependendo dos recursos e do tamanho da rede, um ou alguns sensores são implantados para monitorar os principais ativos ou caminhos de rede.

## Sistemas de detecção de intrusão

O tráfego capturado por cada sensor é transferido para um host ou dispositivo que executa um [[sistema de detecção de intrusão (IDS)]], como Snort ([snort.org](https://www.snort.org/)), Suricata ([suricata-ids.org](https://suricata.io/)) ou Zeek/Bro ([zeek.org](https://zeek.org/)). Quando o tráfego corresponde a uma assinatura de detecção ou a um padrão heurístico, o IDS gera um alerta ou registra uma entrada de log, mas não bloqueia o host de origem. Este tipo de sensor passivo não desacelera o tráfego e não pode ser detectado pelo invasor.

Um IDS é usado para identificar e registrar hosts e aplicativos e para detectar tentativas de descoberta de senhas, varreduras de portas, worms, aplicativos de backdoor, pacotes ou sessões inválidos e outras violações de políticas.

## Sistemas de prevenção de intrusão

[[Sistemas de prevenção de intrusão (IPS)]] são um dispositivo ou software capaz de reagir de forma proativa. Um IPS verifica o tráfego em busca por correspondências a assinaturas de ameaças e pode ser configurado para atuar automaticamente para interromper um ataque. Estas são respostas comuns:

- Bloquear de forma temporária ou permanente a origem do tráfego em desacordo. Isso é chamado de shunning (rejeição).
- Redefinir a conexão, mas não bloquear o endereço de origem.
- Redirecionar o tráfego para um honeypot ou honeynet para análise adicional de ameaças.

Um IPS pode ser implantado como um dispositivo em linha com firewall integrado e capacidade de roteamento/encaminhamento. Se for implantado com um sensor passivo, ele poderá implementar a ação de resposta reconfigurando outro dispositivo, como um firewall ou roteador. Essa funcionalidade usa um script ou uma interface de programação de aplicativo (API) para integrar os dois controles de segurança.

Nota:

Observação: Tanto o Snort quanto o Suricata podem ser configurados como sistemas de prevenção de intrusão.
