

Um [[balanceador de carga]] distribui solicitações de cliente entre os nós de servidor disponíveis em uma farm ou pool. Ele é usado para fornecer serviços que podem escalar de cargas leves a pesadas e para mitigar ataques de negação de serviço. Balanceadores de carga também fornecem tolerância a falhas. Se houver vários servidores disponíveis em uma farm, todos endereçados por um único nome/endereço IP através de um balanceador de carga, se um único servidor falhar, as solicitações de clientes poderão ser encaminhadas a outro servidor na farm. Um balanceador de carga pode ser implantado em qualquer situação em que haja vários servidores proporcionando a mesma função. Alguns exemplos são: servidores Web, servidores de e-mail front-end, webconferências, videoconferências e servidores de streaming de mídia.

Existem dois tipos principais de balanceadores de carga:

- **Balanceador de carga de camada 4**: os balanceadores de carga básicos determinam os encaminhamentos com base em endereço IP e valores de porta TCP/UDP, trabalhando na camada de transporte do modelo OSI.
- **Balanceador de carga de camada 7 (switch de conteúdo)**: à medida que os aplicativos Web se tornam mais complexos, os balanceadores de carga modernos precisam ser capazes de determinar os encaminhamentos com base em dados do aplicativo, por exemplo, uma solicitação de um localizador padrão de recursos (URL) específico ou tipos de dados como streaming de vídeo ou áudio. Isso requer uma lógica mais complexa, mas o poder de processamento dos dispositivos modernos é suficiente para lidar com isso.

Descrição

Os três passos são os seguintes:

1. O cliente se conecta ao servidor virtual pela Internet.

2. O balanceador de carga seleciona uma instância do servidor Web para manter a conexão e encaminha o tráfego.

3. Um mecanismo persistente pode manter o cliente conectado ao mesmo servidor, se disponível.

## Agendamento

O algoritmo de agendamento é o código e as métricas que determinam qual nó será selecionado para processar cada solicitação recebida. O tipo mais simples de agendamento é chamado de round-robin, o que significa escolher o próximo nó. Outros métodos incluem escolher o nó com o menor número de conexões ou o melhor tempo de resposta. Cada método pode ser ponderado usando as preferências definidas pelo administrador e/ou informações de carga dinâmica.

O balanceador de carga também deve usar algum tipo de sonda de heartbeat ou de verificação de integridade para determinar se cada nó está disponível e sob carga. Os balanceadores de carga de camada 4 só podem fazer testes básicos de conectividade, enquanto os dispositivos de camada 7 podem testar o estado do aplicativo e verificar a disponibilidade do host.

## Afinidade de IP de origem e persistência de sessões

Quando um dispositivo cliente estabelece uma sessão com um nó específico na farm de servidores, pode ser necessário continuar usando essa conexão durante a sessão. O IP de origem ou a [[afinidade de sessão]] é uma abordagem de camada 4 para lidar com as sessões de usuário. Isso significa que, quando um cliente estabelece uma sessão, ele fica preso ao nó que aceitou primeiro a solicitação.

Um balanceador de carga de camada de aplicativo usa a [[persistência]] para manter um cliente conectado a uma sessão. Geralmente, a persistência funciona configurando um cookie, seja no nó ou injetado pelo balanceador de carga. Esse método pode ser mais confiável do que a afinidade de IP de origem, mas requer que o navegador aceite o cookie.
