

Um canal de gerenciamento de acesso remoto refere-se ao caso específico do uso de uma rede segura e aplicação de shell para administrar um switch de rede, roteador, firewall ou servidor. As estações de trabalho administrativas seguras (SAWs) usadas como clientes de gerenciamento devem ser rigorosamente protegidas, de preferência instaladas apenas com o software necessário para acessar o canal administrativo, por exemplo, um navegador da Web mínimo, cliente de área de trabalho remota ou terminal virtual SSH. As SAWs devem ter acesso negado à Internet ou restrito a alguns sites de fornecedores aprovados para obtenção de patches, drivers e suporte. Os dispositivos também devem ser submetidos a um controle de acesso rigoroso e auditoria para que qualquer uso indevido seja detectado o quanto antes.

## Gerenciamento fora de banda

Os métodos de [[gerenciamento remoto podem ser in-band ou out-of-band (OOB)]]. Um link de gerenciamento em banda compartilha tráfego com outras comunicações na rede de produção. O método de conexão deve usar sessões criptografadas por TLS, IPsec, RDP ou SSH para garantir a confidencialidade e a integridade.

Uma porta de modem ou console de serial em um roteador é um método de gerenciamento fisicamente fora de banda. Também é possível gerenciar um dispositivo de rede usando uma interface baseada em navegador ou um terminal virtual por Ethernet e IP. Esse tipo de link de gerenciamento é feito fora de banda, seja conectando a porta usada para acesso de gerenciamento a uma infraestrutura de rede fisicamente separada ou conectando a uma VLAN de gerenciamento dedicada. A implementação desse método pode ser onerosa, mas o gerenciamento fora de banda é mais seguro e o acesso ao dispositivo é preservado quando há problemas que afetam a rede de produção.

## Servidores Jump

Um dos desafios do gerenciamento de hosts expostos à Internet, como em uma sub-rede filtrada ou rede em nuvem, é fornecer acesso administrativo aos servidores e dispositivos localizados nela. Por um lado, é necessário um link; por outro, a interface administrativa pode ser comprometida e explorada como um access point para o restante da rede. Consequentemente, o gerenciamento de hosts com permissão para acessar interfaces administrativas em hosts na zona segura deve ser rigorosamente controlado. O processo de configuração e auditoria desse tipo de controle se torna complexo quando há diversos servidores operando na zona.

Uma solução para essa complexidade é adicionar um servidor de administração único, ou [[jump server]], à zona segura. O jump server executa somente a porta administrativa e o protocolo necessários, como SSH ou RDP. Os administradores se conectam ao jump server e, em seguida, o utilizam para se conectar à interface de administração no servidor de aplicativos. A interface de administração do servidor de aplicativos tem uma única entrada em sua ACL (o jump server) e nega tentativas de conexão de qualquer outro host.

Descrição

Os primeiros passos são:

1. Uma VPN pode ser usada para acessar o jump server, que autoriza e encaminha o tráfego de gerenciamento para o servidor do aplicativo.

2. O tráfego de gerenciamento de hosts autorizados na rede também pode ser roteado por meio do jump server.

3. Tráfego de gerenciamento de hosts não autorizados é proibido.

4. Os servidores de aplicativos aceitam apenas conexões de tráfego de gerenciamento do jump server.

5. O tráfego de aplicativos comum utiliza uma rede diferente.
