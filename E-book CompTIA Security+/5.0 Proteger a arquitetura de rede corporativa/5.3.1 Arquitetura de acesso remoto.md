# Security+ CertMaster Study (Versão em português) 7.0

Na rede de [[acesso remoto]], o dispositivo do usuário não faz uma conexão direta com a rede via cabo ou wireless. A conexão ocorre por meio de uma rede intermediária.

Antigamente, o acesso remoto usava modems analógicos conectados pelo sistema de telefonia. Atualmente, a maioria dos acessos remotos é implementada como uma [[rede privada virtual (VPN)]], operando sobre redes de provedores de serviços de Internet (ISP).

Com uma VPN de acesso remoto, os clientes se conectam a um gateway de VPN na borda da rede privada. Essa topologia de VPN cliente a provedor (client-to-site) é o modelo “telecomutador”, que permite que trabalhadores em domicílio e funcionários que atuam em campo se conectem à rede corporativa. O protocolo VPN estabelece um [[túnel]] seguro para manter o conteúdo privado, mesmo quando os pacotes passam pelos roteadores dos ISPs.

Descrição

A sub-rede filtrada de acesso remoto é composta por um gateway VPN e um roteador ou firewall, e a rede local é composta por um roteador ou firewall e servidores LAN. Os primeiros passos são:

1. O host cliente VPN conecta-se a um gateway VPN utilizando qualquer tipo de método de acesso de assinante à Internet.

2. O gateway VPN autentica o usuário e cria um túnel seguro e criptografado.

3. O tráfego do cliente VPN é roteado pela rede e pode acessar os serviços autorizados.

Uma VPN também pode ser implantada em um modelo site a site para conectar duas ou mais redes privadas. Enquanto as conexões VPN de acesso remoto são normalmente iniciadas pelo cliente, uma VPN site a site é configurada para operar automaticamente. Os gateways trocam informações de segurança usando qualquer protocolo de base da VPN. Isso estabelece uma relação de confiança entre os gateways e configura uma conexão segura para o túnel de dados. Os hosts em cada local não precisam ser configurados com informações sobre a VPN. A infraestrutura de roteamento em cada local determina se o tráfego deve ser entregue localmente ou enviado pelo túnel VPN.

Descrição

A VPN site-to-site é composta por três componentes. Um escritório filial que consiste nos servidores e clientes da filial, além do roteador, firewall ou gateway VPN. Uma sub-rede filtrada do escritório matriz composta por gateway VPN e roteador ou firewall. Uma rede local do escritório matriz composta por roteador ou firewall e servidores do escritório matriz. Os primeiros passos são:

1. O gateway de VPN em uma filial estabelece uma conexão VPN com o local da matriz.

2. O tráfego para um host em um local remoto é roteado e encapsulado automaticamente pelo link VPN.

Nota:

Uma terceira topologia é o **túnel host-to-host**. Este é um meio de proteger o tráfego entre dois computadores quando a rede privada não é confiável.

Diversos protocolos de VPN já foram usados ao longo dos anos. Protocolos legados, como o [[Point-to-Point Tunneling Protocol (PPTP)]], foram descontinuados por não oferecerem segurança adequada. Agora, o Transport Layer Security (TLS) (segurança da camada de transporte) e o Internet Protocol Security (IPsec) (protocolo de segurança IP) são as opções preferenciais para configurar o acesso à VPN.
