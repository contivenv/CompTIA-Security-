# 

[[Tecnologias de deception e disruption]] são ferramentas e técnicas de resiliência em cibersegurança que visam aumentar o custo do planejamento de ataques para o ator de ameaça. Honeypots, honeynets, honeyfiles e honeytokens são ferramentas de cibersegurança utilizadas para detectar e defender contra ataques. [[Honeypots]] são sistemas de chamariz que imitam sistemas e aplicações reais. Eles são projetados para permitir que as equipes de segurança monitorem as atividades dos invasores e coletem informações sobre suas táticas e ferramentas. As honeynets são uma rede de honeypots interconectados que simulam uma rede inteira, proporcionando um ambiente mais extenso e realista para os invasores se envolverem. Honeyfiles são arquivos falsos que parecem conter informações confidenciais, usados para detectar tentativas de acesso e roubo de dados. Honeytokens são credenciais falsas, credenciais de login ou outros tipos de dados usados para distrair os invasores, disparar alertas e fornecer informações sobre a atividade do invasor.

Ao implementar essas ferramentas, as organizações podem detectar e monitorar ataques, coletar informações sobre os invasores e seus métodos e se defender proativamente contra ataques futuros. Essas ferramentas também podem fornecer uma camada adicional de defesa, desviando a atenção dos invasores de sistemas e aplicativos reais, reduzindo o risco de ataques bem-sucedidos.

## Estratégias de interrupção

Outro tipo de defesa ativa usa estratégias de interrupção. Elas adotam algumas das estratégias de ofuscação usadas por atores maliciosos. O objetivo é aumentar o custo do ataque e limitar os recursos do adversário. Alguns exemplos de estratégias de interrupção incluem o seguinte:

- Usar entradas DNS falsas para listar vários hosts que não existem.
- Configurar um servidor Web com vários diretórios de chamariz ou páginas geradas dinamicamente para retardar a varredura.
- Utilizar port triggering ou spoofing para retornar dados de [[telemetria falsos]] quando um host detecta atividade de varredura de portas. Isso resultará em várias portas sendo falsamente relatadas como abertas e diminuindo a velocidade da varredura. A telemetria pode se referir a qualquer tipo de medição ou dados retornados por varredura remota. Telemetria falsa semelhante pode ser usada para relatar endereços IP como ativos quando eles não o estão, por exemplo.
- Utilizar um [[DNS sinkhole]] para direcionar o tráfego suspeito para uma rede diferente, como uma honeynet, onde ele pode ser analisado.
